# 修复界面申请人字段显示修复说明

## 📅 修改日期
2026年4月24日

---

## 🐛 问题描述

修复界面的申请人字段显示为空。

**原因：**
- 数据库字段已从 `applicant` (VARCHAR) 改为 `applicant_id` (BIGINT)
- 后端查询时通过 JOIN `sys_user` 表返回 `applicantName` 字段
- 前端代码仍然使用 `applicant` 字段，导致显示为空

---

## 🔧 解决方案

### 前端修改

**文件：** `frontend/src/views/RepairsView.vue`

#### 1. 修复表格列显示

**修改前：**
```vue
<el-table-column prop="applicant" :label="$t('repair.applicant')" width="100" />
```

**修改后：**
```vue
<el-table-column prop="applicantName" :label="$t('repair.applicant')" width="100" />
```

---

#### 2. 修复详情对话框显示

**修改前：**
```vue
<el-descriptions-item :label="$t('repair.applicant')">
  {{ currentDetail.applicant }}
</el-descriptions-item>
```

**修改后：**
```vue
<el-descriptions-item :label="$t('repair.applicant')">
  {{ currentDetail.applicantName }}
</el-descriptions-item>
```

---

## 📊 数据流程

### 完整的数据流程

```
1. 用户提交修复申请
   ↓
2. Controller 获取 username
   ↓
3. Service 通过 username 查询 userId
   ↓
4. 保存 applicant_id 到数据库
   ↓
5. 查询时 JOIN sys_user 表
   ↓
6. 返回 applicantName (real_name)
   ↓
7. 前端显示 applicantName
```

---

### 后端查询SQL

```sql
SELECT 
    rr.*, 
    cr.relic_name, 
    cr.relic_code, 
    su.real_name as applicant_name
FROM repair_record rr
LEFT JOIN cultural_relic cr ON rr.relic_id = cr.id
LEFT JOIN sys_user su ON rr.applicant_id = su.id
```

---

### 后端返回的数据结构

```json
{
  "id": 1,
  "repairCode": "REP2026001",
  "relicId": 8,
  "relicName": "唐代银壶",
  "status": "待审批",
  "priority": "高",
  "applicantId": 1,           // 申请人ID
  "applicantName": "张三",     // 申请人姓名（用于显示）
  "applyDate": "2026-04-24 10:00:00",
  "repairReason": "表面氧化严重",
  ...
}
```

## 📋 相关字段对比

|--------|--------|------|------|
| applicant | applicant_id | BIGINT | 申请人ID（外键关联sys_user.id） |
