# 数据备份恢复系统 - 快速开始

## 安装步骤

### 1. 执行数据库脚本

```bash
# 在MySQL中执行
mysql -u root -p < backend/sql/backup_system.sql
```

这将创建以下表：
- `sys_backup` - 备份记录表
- `sys_backup_config` - 备份配置表
- `sys_restore` - 恢复记录表

### 2. 确认数据库权限

确保应用使用的数据库用户有以下权限：
```sql
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, RELOAD, LOCK TABLES ON *.* TO 'your_user'@'localhost';
FLUSH PRIVILEGES;
```

### 3. 确认系统命令

确保系统中可以执行以下命令：
```bash
# 测试mysqldump
mysqldump --version

# 测试mysql
mysql --version
```

### 4. 创建备份目录

```bash
# 在项目根目录创建备份目录
mkdir -p backups
chmod 755 backups
```

### 5. 启动应用

```bash
# 后端
cd backend
mvn spring-boot:run

# 前端
cd frontend
npm run dev
```

## 快速使用

### 1. 访问备份管理页面

登录系统后，点击左侧菜单"数据备份"

### 2. 创建第一个备份

1. 点击"创建备份"按钮
2. 输入备份名称：`测试备份`
3. 输入描述：`第一次手动备份`
4. 选择是否加密（建议先不加密测试）
5. 点击确定

系统将在后台执行备份，刷新页面查看备份状态。

### 3. 下载备份文件

1. 等待备份状态变为"成功"
2. 点击"下载"按钮
3. 浏览器将下载备份文件

### 4. 配置自动备份

1. 点击"备份配置"按钮
2. 启用自动备份开关
3. 选择备份频率：每天
4. 设置备份时间：02:00
5. 设置保留天数：30
6. 设置最大备份数：10
7. 点击确定保存

### 5. 测试数据恢复

⚠️ **警告**：恢复操作会覆盖当前数据库，请谨慎操作！

1. 在测试环境中，先创建一个备份
2. 修改一些数据（用于验证恢复效果）
3. 点击备份的"恢复"按钮
4. 确认恢复操作
5. 等待恢复完成
6. 验证数据是否恢复到备份时的状态

## 常见问题

### Q1: 备份一直处于"处理中"状态？

**原因**：
- mysqldump命令执行失败
- 数据库权限不足
- 磁盘空间不足

**解决方法**：
1. 检查后端日志查看错误信息
2. 手动执行mysqldump命令测试
3. 检查数据库用户权限
4. 检查磁盘空间

### Q2: 恢复失败？

**原因**：
- mysql命令执行失败
- 备份文件损坏
- 数据库权限不足

**解决方法**：
1. 检查后端日志查看错误信息
2. 手动执行mysql命令测试
3. 验证备份文件完整性
4. 检查数据库用户权限

### Q3: 加密备份无法下载？

**原因**：
- 解密失败
- 密钥不匹配

**解决方法**：
1. 检查AES_KEY配置
2. 确保加密和解密使用相同的密钥
3. 查看后端日志

### Q4: Windows系统下命令执行失败？

**原因**：
- Windows和Linux命令格式不同
- PATH环境变量未配置

**解决方法**：
1. 确保MySQL bin目录在PATH中
2. 修改BackupService中的命令格式
3. 使用绝对路径执行命令

### Q5: 自动备份不执行？

**原因**：
- 定时任务未启用
- 配置未保存
- 时间设置错误

**解决方法**：
1. 检查备份配置是否启用
2. 确认备份时间设置正确
3. 查看后端日志
4. 检查Spring的@Scheduled配置

## 性能参考

### 备份时间
- 小型数据库（< 100MB）：10-30秒
- 中型数据库（100MB - 1GB）：1-5分钟
- 大型数据库（> 1GB）：5分钟以上

### 恢复时间
- 通常比备份时间稍长
- 取决于数据库大小和服务器性能

### 磁盘空间
- 每个备份约为数据库大小的80-120%
- 建议预留至少10个备份的空间

## 生产环境建议

### 1. 备份策略
- 每天凌晨2点自动备份
- 保留最近30天的备份
- 每周日创建一个长期备份（手动）
- 重要操作前手动备份

### 2. 安全配置
- 启用备份文件加密
- 使用密钥管理系统
- 限制备份文件访问权限
- 定期测试恢复流程

### 3. 监控告警
- 监控备份成功率
- 监控磁盘空间使用
- 备份失败时发送告警
- 定期验证备份文件

### 4. 异地备份
- 将备份文件同步到其他服务器
- 使用云存储服务
- 定期下载备份到本地
