## 常见问题排查

### 问题1：仍然产生两条日志
**可能原因：**
- 后端服务未重启
- 缓存未清除

**解决方案：**
```bash
# 停止后端服务
Ctrl+C

# 清理并重新编译
cd backend
mvn clean compile

# 重启服务
mvn spring-boot:run
```

### 问题2：日志详情仍然没有数据对比
**可能原因：**
- 数据库字段未添加（before_data、after_data、changed_fields）
- 前端缓存未清除

**解决方案：**
```bash
# 1. 检查数据库字段
mysql -u root -p cultural_relics -e "DESC sys_operation_log;"

# 2. 如果字段缺失，执行修复脚本
mysql -u root -p cultural_relics < backend/sql/audit_log_fix_missing_fields.sql

# 3. 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete → 清除缓存
# 或使用无痕模式测试
```

### 问题3：操作状态显示"失败"
**可能原因：**
- 业务操作本身失败
- 审计日志记录异常

**解决方案：**
1. 查看后端控制台日志
2. 检查是否有异常堆栈信息
3. 确认业务操作是否成功执行
