## 使用说明

### 启动系统

```bash
# 前端
cd frontend
npm run dev

# 后端（如果未启动）
cd backend
mvn spring-boot:run
```

### 访问页面

1. **员工管理**
   - URL: http://localhost:5173/employees
   - 权限：需要 `users:manage` 权限

2. **借展人管理**
   - URL: http://localhost:5173/loaners
   - 权限：需要 `users:manage` 权限

3. **博物馆管理**
   - URL: http://localhost:5173/museums
   - 权限：需要 `users:manage` 权限

### 操作流程

#### 新增员工
1. 点击"新增员工"按钮
2. 填写用户名、密码、真实姓名
3. 选择角色（系统管理员/文物保管员/借展审批员）
4. 填写电话、邮箱
5. 选择状态（启用/禁用）
6. 点击"确定"保存

#### 新增借展人
1. 点击"新增借展人"按钮
2. 填写用户名、密码、真实姓名
3. **选择所属博物馆**（必填）
4. 填写电话、邮箱
5. 选择状态
6. 点击"确定"保存

#### 新增博物馆
1. 点击"新增博物馆"按钮
2. 填写博物馆编码、名称
3. 选择博物馆类型
4. 选择合作状态（有合作/无合作）
5. 填写省份、城市、地址
6. 填写联系人、联系电话、联系邮箱
7. 填写描述（可选）
8. 点击"确定"保存

## 与原用户管理的区别

### 员工管理 vs 用户管理
- **员工管理**：只显示系统管理员、文物保管员、借展审批员
- **用户管理**：显示所有用户（包括借展人）

### 借展人管理 vs 用户管理
- **借展人管理**：只显示借展人，必须关联博物馆
- **用户管理**：显示所有用户，借展人可选择博物馆

### 优势
- 职责分离，界面更清晰
- 针对性操作，减少误操作
- 借展人管理强制关联博物馆

## 注意事项

1. **删除操作**
   - 不能删除当前登录用户
   - 博物馆删除为逻辑删除（status改为0）
   - 用户删除为物理删除

2. **密码修改**
   - 编辑用户时，密码留空表示不修改
   - 新增用户时，密码必填

3. **博物馆关联**
   - 新增借展人时必须选择博物馆
   - 编辑借展人时可以更改博物馆
   - 删除博物馆前应检查是否有关联的借展人

4. **权限控制**
   - 所有三个页面都需要 `users:manage` 权限
   - 只有系统管理员可以访问
