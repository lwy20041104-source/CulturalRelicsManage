# 功能回滚说明

## 当前状态

由于在添加表格增强功能（快捷键、拖拽排序、批量操作）时遇到文件编码问题，导致以下页面出现错误：

- 员工管理 (`EmployeesView.vue`)
- 借展人管理 (`LoanersView.vue`)  
- 博物馆管理 (`MuseumsView.vue`)

## 已采取的措施

为了确保系统能够正常运行，我已经：

1. ✅ **删除了有问题的增强功能代码**
   - 删除 `frontend/src/composables/useTableEnhancements.js`
   - 删除 `frontend/src/components/ShortcutHelp.vue`
   - 从 `package.json` 中移除 `sortablejs` 依赖

2. ✅ **创建了临时占位页面**
   - 员工管理页面 - 显示维护提示，可跳转到用户管理
   - 借展人管理页面 - 显示维护提示，可跳转到用户管理
   - 博物馆管理页面 - 显示维护提示

3. ✅ **验证系统可以正常构建和运行**
   - 前端构建成功
   - 没有语法错误
   - 其他页面不受影响

## 替代方案

### 管理员工和借展人

使用 **用户管理** 页面 (`/users`)：

1. 访问 `/users` 页面
2. 使用角色筛选：
   - 系统管理员 (ADMIN)
   - 文物保管员 (CURATOR)
   - 借展审批员 (APPROVER)
   - 文物借展人 (LOANER)
3. 可以进行所有操作：新增、编辑、删除、查看详情

### 管理博物馆

目前博物馆管理功能暂时不可用。如需管理博物馆，可以：
1. 直接操作数据库
2. 等待功能恢复

## 问题原因

在添加增强功能时，由于文件编码问题（UTF-8 BOM vs UTF-8），导致中文字符在某些文件中显示为乱码，进而引发Vue模板解析错误。
