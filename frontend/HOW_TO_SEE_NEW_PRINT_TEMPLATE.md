# 如何查看新的打印模板

## 问题
打印模板已经更新，但浏览器可能显示的是旧版本（缓存问题）。

## 解决方案

### 🔥 最快方法：硬刷新浏览器

在浏览器中按以下快捷键：

#### Windows 用户
```
Ctrl + Shift + R
或
Ctrl + F5
```

#### Mac 用户
```
Cmd + Shift + R
```

### 📋 详细步骤

1. **打开档案管理页面**
   - 登录系统
   - 点击左侧菜单"档案管理"

2. **硬刷新浏览器**
   - 按 `Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)
   - 等待页面完全重新加载

3. **测试打印功能**
   - 点击任意档案的"更多"按钮
   - 选择"打印"
   - 查看打印预览对话框

4. **验证新模板**
   - ✅ 应该看到"档案详情"标题
   - ✅ 标题下方显示"打印时间: YYYY-MM-DD HH:mm:ss"
   - ✅ 信息以表格形式显示
   - ✅ 表格有边框，标签列灰色背景

## 新模板 vs 旧模板

### 旧模板特征
- 使用 Element Plus 的 Descriptions 组件
- 蓝色下划线
- 没有打印时间

### 新模板特征
- 使用 HTML 表格
- 黑色边框
- 标签列灰色背景
- **显示打印时间**

## 如果还是看不到新模板

### 方法1：清除浏览器缓存
1. 按 `F12` 打开开发者工具
2. 右键点击浏览器刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方法2：使用无痕模式
1. 按 `Ctrl + Shift + N` (Chrome) 或 `Ctrl + Shift + P` (Firefox)
2. 在无痕窗口中访问系统
3. 测试打印功能

### 方法3：重启开发服务器
```bash
# 在终端中按 Ctrl + C 停止服务器
# 然后重新启动
cd frontend
npm run dev
```

### 方法4：检查文件是否更新
```bash
# 在项目根目录执行
grep -n "print-header" frontend/src/views/ArchivesView.vue
grep -n "currentPrintTime" frontend/src/views/ArchivesView.vue
```

如果看到这些行，说明文件已更新：
- `<div class="print-header">`
- `{{ currentPrintTime }}`

## 验证清单

打开打印预览后，应该看到：

- [ ] 标题："档案详情"（居中）
- [ ] 打印时间：显示在标题下方
- [ ] 档案基本信息：表格形式，4列
- [ ] 关联文物信息：表格形式
- [ ] 档案描述：灰色背景框
- [ ] 档案管理信息：表格形式
- [ ] 附件信息：如果有附件

点击"打印"按钮后，在浏览器打印预览中：

- [ ] 只显示档案信息
- [ ] 没有菜单栏
- [ ] 没有按钮
- [ ] 没有对话框边框
- [ ] 表格边框清晰可见

## 仍然有问题？

请检查浏览器控制台（F12）是否有错误信息，并提供：
1. 浏览器类型和版本
2. 控制台错误信息截图
3. 当前看到的打印预览截图

---

**重要提示**：如果使用的是 Chrome 浏览器，硬刷新（Ctrl + Shift + R）通常可以立即解决缓存问题。
