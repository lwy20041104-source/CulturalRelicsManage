# 批量操作功能说明

## 功能概述

为员工管理、借展人管理和博物馆管理三个页面添加了批量操作功能，提升管理效率。

## 实现的功能

### 1. 多选功能
- ✅ 表格第一列添加了多选框
- ✅ 可以选择单个或多个记录
- ✅ 支持全选/取消全选

### 2. 批量删除
- ✅ 选中记录后，点击"批量删除"按钮
- ✅ 显示确认对话框，提示删除数量
- ✅ 员工/借展人管理：自动检查是否包含当前登录用户，防止误删
- ✅ 删除成功后自动刷新列表并清空选择

### 3. 批量修改状态
- ✅ 选中记录后，点击"批量修改状态"按钮
- ✅ 弹出对话框选择新状态
  - 员工/借展人：启用/禁用
  - 博物馆：有合作/无合作
- ✅ 修改成功后自动刷新列表并清空选择

## 使用说明

### 员工管理 (`/employees`)

**批量删除：**
1. 勾选需要删除的员工
2. 点击"批量删除"按钮
3. 确认删除操作
4. 系统自动删除并刷新列表

**批量修改状态：**
1. 勾选需要修改的员工
2. 点击"批量修改状态"按钮
3. 在弹出框中选择"启用"或"禁用"
4. 确认后系统自动更新状态

**注意事项：**
- 不能删除当前登录用户
- 批量操作会检查是否包含当前登录用户

### 借展人管理 (`/loaners`)

**批量删除：**
1. 勾选需要删除的借展人
2. 点击"批量删除"按钮
3. 确认删除操作
4. 系统自动删除并刷新列表

**批量修改状态：**
1. 勾选需要修改的借展人
2. 点击"批量修改状态"按钮
3. 在弹出框中选择"启用"或"禁用"
4. 确认后系统自动更新状态

**注意事项：**
- 不能删除当前登录用户
- 批量操作会检查是否包含当前登录用户

### 博物馆管理 (`/museums`)

**批量删除：**
1. 勾选需要删除的博物馆
2. 点击"批量删除"按钮
3. 确认删除操作
4. 系统自动删除（逻辑删除）并刷新列表

**批量修改状态：**
1. 勾选需要修改的博物馆
2. 点击"批量修改状态"按钮
3. 在弹出框中选择"有合作"或"无合作"
4. 确认后系统自动更新状态

**注意事项：**
- 博物馆删除为逻辑删除（将status改为0）
- 删除前应检查是否有关联的借展人

```javascript

const handleSelectionChange = (selection) => {
  selectedIds.value = selection.map(item => item.id)
}
```

**3. 批量删除**
```javascript
const batchDelete = async () => {
  await ElMessageBox.confirm(`确定要删除选中的 ${selectedIds.value.length} 个记录吗？`, '警告', { type: 'warning' })
  
  try {
    await Promise.all(selectedIds.value.map(id => deleteApi(id)))
    ElMessage.success('批量删除成功')
    selectedIds.value = []
    loadData()
  } catch (e) {
    ElMessage.error('批量删除失败')
  }
}
```

### 按钮状态控制

```vue
<el-button type="danger" :disabled="selectedIds.length === 0" @click="batchDelete">
  批量删除
</el-button>
```

## 用户体验优化

### 1. 按钮禁用状态
- 未选择记录时，批量操作按钮自动禁用
- 避免用户误操作

### 2. 确认对话框
- 所有批量操作都有确认对话框
- 显示操作数量，让用户明确操作范围

### 3. 操作反馈
- 操作成功：显示成功提示
- 操作失败：显示错误提示
- 自动刷新列表

### 4. 安全检查
- 员工/借展人管理：防止删除当前登录用户
- 博物馆管理：逻辑删除，保留数据

## 性能优化

### 1. 并发请求
使用 `Promise.all()` 并发执行多个删除/更新请求，提升操作速度。

```javascript
await Promise.all(selectedIds.value.map(id => deleteApi(id)))
```

### 2. 操作后清空选择
操作完成后自动清空选择状态，避免重复操作。

```javascript
selectedIds.value = []
```
