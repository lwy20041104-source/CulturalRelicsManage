## 🎯 使用场景

### 场景1：管理员日常工作

**早上上班**：
1. 登录系统
2. 看到通知铃铛显示 🔔(5)
3. 点击铃铛查看通知
4. 发现3个借展申请、1个修复申请、1个逾期提醒
5. 点击借展申请通知，跳转到借展管理页面
6. 处理借展申请
7. 通知自动标记为已读

---

### 场景2：借展人申请借展

**提交申请**：
1. 登录系统（借展人账号）
2. 进入借展管理页面
3. 填写借展申请并提交
4. **系统自动发送通知给管理员和审批员**

**等待审批**：
1. 审批员审批申请
2. **系统自动发送审批结果通知给借展人**
3. 借展人看到通知铃铛显示 🔔(1)
4. 点击查看，发现申请已通过
5. 点击通知跳转到借展管理页面查看详情

---

### 场景3：逾期提醒

**自动检查**：
1. 每天凌晨1点，系统自动检查所有借展记录
2. 发现有文物逾期未归还
3. **系统自动发送逾期通知给管理员和保管员**

**管理员处理**：
1. 早上登录系统
2. 看到逾期通知
3. 点击通知查看详情
4. 联系借展人催还文物

## ⚙️ 系统设置

### 定时任务配置

**逾期检查时间**：每天凌晨1点

**修改方法**（如需调整）：
```java
// 文件：backend/src/main/java/com/example/task/LoanOverdueCheckTask.java

// 当前配置：每天凌晨1点
@Scheduled(cron = "0 0 1 * * ?")

// 改为每天上午9点
@Scheduled(cron = "0 0 9 * * ?")

// 改为每小时执行一次
@Scheduled(cron = "0 0 * * * ?")

// 改为每30分钟执行一次
@Scheduled(cron = "0 */30 * * * ?")
```

## 🔍 常见问题

### Q1：为什么没有收到通知？

**可能原因**：
1. 用户角色不匹配（例如：借展申请通知只发给管理员和审批员）
2. 用户状态为禁用
3. 后端服务未启动
4. 数据库连接失败

**解决方法**：
1. 检查用户角色是否正确
2. 检查用户状态是否为"启用"
3. 查看后端日志是否有错误
4. 测试数据库连接

---

### Q2：通知铃铛不显示？

**可能原因**：
1. 前端服务未启动
2. 浏览器缓存问题
3. 网络请求失败

**解决方法**：
1. 确认前端服务正常运行
2. 清除浏览器缓存并刷新
3. 打开浏览器控制台查看错误信息
4. 检查网络请求是否成功

---

### Q3：通知延迟太大？

**当前方式**：轮询（每30秒刷新）

**解决方法**：
1. 调整轮询间隔（改为10秒或15秒）
2. 或者安装WebSocket依赖启用实时推送（0延迟）

**启用WebSocket**：
```bash
cd frontend
npm install sockjs-client @stomp/stompjs
```

然后参考 `frontend/WEBSOCKET_SETUP.md` 文档完成配置。

---

### Q4：定时任务没有执行？

**检查方法**：
1. 查看后端日志：
```bash
grep "开始检查逾期借展记录" logs/application.log
```

2. 确认Application类有@EnableScheduling注解（已确认有）

3. 检查服务器时间是否正确

4. 验证Cron表达式是否正确

---

### Q5：如何查看通知统计？

**方法1**：通知列表页面
1. 点击通知铃铛底部的"查看全部"
2. 进入通知列表页面
3. 点击右下角的统计按钮（📊图标）
4. 查看详细统计信息

**方法2**：API接口
```bash
curl http://localhost:8080/api/notifications/statistics
```
