# 百度AI图像识别集成文档

## 概述

本文档说明如何在博物馆文物数字化管理系统中集成百度AI图像识别服务，以提升文物自动分类的准确率。

## 集成时间

**2026年5月6日** - 完成百度AI图像识别集成

## 功能特性

### 1. 混合识别策略
- **百度AI识别**：优先使用百度AI进行识别（准确率85-92%）
- **规则识别**：AI不可用时自动回退到规则识别（准确率40-50%）
- **混合模式**：AI置信度较低时结合规则识别（准确率60-75%）

### 2. 智能回退机制
- 百度AI未启用：自动使用规则识别
- 百度AI配置错误：自动使用规则识别
- 百度AI调用失败：自动使用规则识别
- 百度AI置信度低：结合规则识别补充

### 3. 关键词映射系统
- **80+个文物关键词**到分类的映射
- 支持精确匹配和模糊匹配
- 涵盖10种文物分类

## 技术实现

### 1. 依赖添加

在 ackend/pom.xml 中添加百度AI SDK：

\\\xml
<!-- Baidu AI SDK for image recognition -->
<dependency>
    <groupId>com.baidu.aip</groupId>
    <artifactId>java-sdk</artifactId>
    <version>4.16.18</version>
</dependency>
\\\

### 2. 配置文件

在 ackend/src/main/resources/application.yml 中添加配置：

\\\yaml
# 百度AI配置
baidu:
  ai:
    # 是否启用百度AI识别（false则使用规则识别）
    enabled: false
    # 百度AI应用的APP_ID
    app-id: your_app_id_here
    # 百度AI应用的API_KEY
    api-key: your_api_key_here
    # 百度AI应用的SECRET_KEY
    secret-key: your_secret_key_here
    # 识别置信度阈值（0-1之间，低于此值将使用规则识别作为补充）
    confidence-threshold: 0.6
\\\

### 3. 核心类

#### BaiduAiConfig.java
配置类，负责初始化百度AI客户端。

#### BaiduAiRecognitionServiceImpl.java
百度AI识别服务实现，包含：
- 80+个关键词到分类的映射
- 百度AI接口调用
- 结果转换和评分

#### RelicImageRecognitionServiceImpl.java
混合识别策略实现，包含：
- 优先使用百度AI识别
- 智能回退到规则识别
- 结果合并和优化

## 使用方法

### 1. 申请百度AI密钥

1. 访问百度AI开放平台：https://ai.baidu.com/
2. 注册并登录账号
3. 创建应用，选择"图像识别"
4. 获取APP_ID、API_KEY、SECRET_KEY

### 2. 配置密钥

修改 pplication.yml：

\\\yaml
baidu:
  ai:
    enabled: true  # 启用百度AI
    app-id: 你的APP_ID
    api-key: 你的API_KEY
    secret-key: 你的SECRET_KEY
    confidence-threshold: 0.6  # 置信度阈值
\\\

### 3. 重启后端服务

\\\ash
cd backend
mvn clean install
mvn spring-boot:run
\\\

### 4. 测试识别功能

1. 登录系统
2. 进入文物管理页面
3. 添加或编辑文物
4. 上传文物图片
5. 点击"AI识别"按钮
6. 查看识别结果

## 故障排查

### 问题1：识别失败，提示"百度AI不可用"
**原因**：
- enabled设置为false
- API密钥未配置或配置错误
- 网络连接问题

**解决方案**：
1. 检查application.yml配置
2. 确认API密钥正确
3. 检查网络连接
4. 查看后端日志

### 问题2：识别准确率低
**原因**：
- 图片质量差
- 文物类型不在支持范围内
- 百度AI未识别到相关物体

**解决方案**：
1. 上传高质量图片
2. 确保文物类型在支持范围内
3. 手动选择分类

### 问题3：识别速度慢
**原因**：
- 网络延迟
- 图片文件过大

**解决方案**：
1. 优化网络环境
2. 压缩图片大小（建议<2MB）
