# 博物馆文物数字化管理系统 - 完整功能与技术文档

## 一、项目概述

博物馆文物数字化管理系统是一个基于 Spring Boot + Vue 3 的全栈 Web 应用，用于博物馆文物的全生命周期管理，包括文物信息管理、借展管理、修复管理、数据分析和 AI 智能查询等功能。

**项目特色**：
- ✅ 27个核心功能模块，覆盖文物管理全流程
- ✅ 完善的安全机制（JWT认证、登录限制、密码重置）
- ✅ 实时通知系统（WebSocket推送、邮件通知）
- ✅ 统一的异常处理和日志规范
- ✅ 真实的邮件和短信服务集成
- ✅ 严格的业务规则验证
- ✅ 丰富的数据可视化展示
- ✅ 主题自定义系统（6个主题 × 2种模式 = 12种视觉风格）
- ✅ 完整的用户管理体系（个人信息、员工管理、借展人管理）
- ✅ 批量操作功能（批量删除、批量修改状态）

### 4. 文物二维码标签生成与扫描 ✨ (新增)

#### 功能特性
- **唯一二维码生成**：为每个文物生成唯一的二维码标签
- **二维码内容**：包含文物编号、名称和扫描URL
- **在线查看**：在文物列表中直接查看二维码
- **下载功能**：将二维码保存为PNG图片文件
- **打印功能**：生成专业的打印预览页面
- **扫描查看**：扫描后跳转到专门的展示页面
- **无需登录**：扫描页面无需登录即可查看基本信息
- **移动端适配**：响应式设计，支持手机扫描

#### 技术实现
- **Google ZXing**：二维码生成库（Java）
  - 版本：3.5.1
  - 支持多种编码格式
  - 高质量图片输出
- **QRCodeUtil 工具类**：
  - `generateRelicQRCodeUrl()`：生成二维码URL
  - `generateQRCodeLabelBase64()`：生成带标签的二维码
  - `generateQRCodeImage()`：生成二维码图片
  - `addTextToImage()`：添加文字标签
- **Base64 编码**：前端直接显示，无需文件存储
- **Canvas API**：前端图片处理
- **独立路由**：`/qrcode/:id` 扫描页面路由

#### 应用场景
- **文物标签管理**：制作实体标签贴在展柜
- **展览导览**：观众扫码自助查看文物信息
- **库房管理**：快速定位和识别文物
- **文物盘点**：扫码记录盘点信息

#### 数据流程
1. 用户点击"二维码"按钮
2. 前端调用 `/relics/{id}/qrcode` 接口
3. 后端生成二维码图片（Base64）
4. 前端显示二维码对话框
5. 用户可下载或打印二维码
6. 扫描二维码跳转到 `/qrcode/{id}` 页面
7. 展示文物基本信息（无需登录）

### 12. AI 智能查询

#### 功能特性
- **智能查询引擎**：
  - 关键词匹配：支持多关键词查询（如"青铜器"、"唐代的文物"）
  - 关键词扩展：自动扩展同义词（硬编码）
  - 多维度匹配：名称、年代、材质、分类、描述等多字段匹配
  - 智能评分算法：基于相关度的排序（0-100分）
  - 模糊匹配：支持部分匹配和近似匹配
  
- **双层检索策略**：
  - **馆藏优先**：优先检索本馆文物数据库
    - 精确匹配：文物名称、编号精确匹配
    - 模糊匹配：年代、材质、分类模糊匹配
    - 描述匹配：文物描述全文检索
    - 相关度排序：按匹配度降序排列
  - **全网兜底**：馆藏无结果时自动触发全网检索
    - 百度搜索：搜索相关文物信息
    - 百度百科：提取百科词条内容
    - 博物馆官网：抓取官方文物信息
    - 智能过滤：过滤无关网站（YouTube、Google Play等）
    
- **图片智能抓取**：
  - **7层提取策略**：
    1. og:image meta标签（社交媒体分享图）
    2. 百度百科特定选择器（.summary-pic）
    3. class="summary-pic"图片
    4. 第一个有效img标签
    5. data-src属性（懒加载图片）
    6. srcset属性（响应式图片）
    7. CSS背景图片
  - **图片验证**：URL格式验证、域名白名单过滤
  - **图片代理**：解决防盗链和跨域问题
  
- **会话管理**：
  - 多轮对话支持：保持上下文连续性
  - 会话历史记录：保存所有查询记录
  - 会话标题自动生成：基于首次查询内容
  - 会话列表管理：查看、删除、重命名会话
  - 用户隔离：每个用户独立的会话空间
  
- **查询结果持久化**：
  - **完整的数据链路**：
    - ai_chat_session：会话信息（用户、标题、创建时间）
    - ai_chat_message：消息记录（用户问题、AI回复）
    - ai_query_result：查询结果详情（文物信息、相关度、来源）
  - **丰富的元数据**：
    - 相关度评分（0-100）
    - 匹配标签（名称匹配、年代匹配等）
    - 来源类型（馆藏/外部）
    - 来源名称（百度百科、博物馆官网等）
  - **历史追溯**：完整保留查询历史，支持回溯分析
  
- **智能推荐**：
  - 相关文物推荐：基于查询结果推荐相似文物
  - 热门查询：统计高频查询关键词
  - 查询建议：根据历史查询提供智能建议
  
- **用户体验优化**：
  - 实时搜索：输入即搜索，无需点击按钮
  - 加载动画：查询过程中显示加载状态
  - 结果高亮：关键词高亮显示
  - 图片预览：点击图片放大查看
  - 响应式设计：适配PC和移动端

#### 技术实现
- **关键词处理**：
  - **简单分词**：按空格、逗号等分隔符分割关键词
  - **同义词扩展**：硬编码的同义词映射
    - 司母戊鼎 ↔ 后母戊鼎
    - 四羊方尊 ↔ 四羊青铜方尊
    - 青铜器 ↔ 铜器
    - 陶瓷 ↔ 陶器 ↔ 瓷器
  - **后缀处理**：自动去除"器"后缀（如"青铜器" → "青铜"）
  
  - **查询流程**：
    ```
    用户输入问题
        ↓
    关键词分割（按空格、逗号）
        ↓
    同义词扩展（硬编码映射）
        ↓
    文物匹配与评分
        ↓
    结果排序
        ↓
    返回结果
    ```
  
- **相关度评分算法**：
  ```java
  总分 = 名称匹配分(40分) + 年代匹配分(24分) + 材质匹配分(24分) 
       + 分类匹配分(20分) + 描述匹配分(12分) + 来源匹配分(10分)
  ```
  - 整句命中：名称整句命中+80分、编号整句命中+70分、描述整句命中+30分
  - 关键词命中：名称+40分、编号+35分、年代+24分、材质+24分、状态+24分、分类+20分、描述+12分、来源+10分
  - 同义词匹配：自动识别同义词并计分
  - 匹配标签：记录命中的字段（如："名称命中：青铜器"、"年代命中：商代"）

### 22. 异常处理与日志规范

#### 功能特性
- **自定义异常类体系**：
  - BusinessException：业务异常（业务规则验证失败）
  - ValidationException：参数验证异常（参数校验失败）
  - ResourceNotFoundException：资源未找到异常（数据不存在）
  - ServiceException：服务异常（外部服务调用失败）
- **全局异常处理器增强**：
  - 统一异常捕获和处理
  - 规范化错误响应格式
  - 分级日志记录（ERROR/WARN）
  - Spring Validation异常处理
  - 数据库唯一键冲突处理
  - 空指针异常处理
  - 非法参数异常处理
  - 非法状态异常处理
- **日志规范**：
  - ERROR级别：系统异常、外部服务失败（需要立即处理）
  - WARN级别：业务异常、资源未找到（预期内的错误）
  - INFO级别：重要业务操作、状态变更
  - DEBUG级别：开发调试信息（生产环境关闭）
- **敏感信息脱敏**：
  - 密码：永远不记录
  - 手机号：显示前3位和后4位（138****5678）
  - 邮箱：显示前2位和域名（ab***@example.com）
- **日志格式规范**：
  - 使用占位符：log.info("用户登录：username={}", username)
  - 关键信息前置：操作类型在前，详细信息在后
  - 统一格式：操作类型：关键信息1=值1, 关键信息2=值2
- **规范文档**：
  - LOGGING_GUIDELINES.md：日志使用规范指南
  - EXCEPTION_HANDLING_GUIDELINES.md：异常处理规范指南
  - CODE_QUALITY_IMPROVEMENTS.md：代码质量改进总结

#### 技术实现
- **异常类设计**：
  - 继承RuntimeException：支持事务回滚
  - 包含错误码：支持国际化
  - 包含上下文信息：便于问题定位
- **全局异常处理器**：
  - @RestControllerAdvice：全局异常处理
  - @ExceptionHandler：针对不同异常类型
  - 统一返回Result对象
  - 日志记录：log.error/log.warn
- **代码迁移**：
  - SysUserServiceImpl：替换RuntimeException为BusinessException/ValidationException
  - PasswordResetServiceImpl：替换RuntimeException为ServiceException/ValidationException
  - 其他Service类：逐步迁移
- **最佳实践**：
  - Service层抛出异常
  - Controller层不需要try-catch
  - 全局异常处理器统一处理
  - 事务管理：@Transactional(rollbackFor = Exception.class)

## 四、核心算法与技术点

### 1. 认证与授权算法

#### JWT Token 生成算法
```java
String token = Jwts.builder()
    .setSubject(username)
    .setIssuedAt(new Date())
    .setExpiration(new Date(System.currentTimeMillis() + expiration))
    .signWith(SignatureAlgorithm.HS512, secret)
    .compact();
```

#### BCrypt 密码加密
- **算法**：Blowfish 加密算法
- **加盐**：每次加密生成随机盐值
- **迭代次数**：默认 10 轮（2^10 = 1024 次）
- **特点**：相同密码每次加密结果不同，防止彩虹表攻击

## 八、技术难点与解决方案

### 1. 图片防盗链问题
**问题**：百度百科图片无法直接显示  
**解决**：实现图片代理服务，设置正确的 Referer 头

### 2. 批量操作性能
**问题**：大量数据批量处理性能差  
**解决**：使用事务批量提交，优化 SQL 语句

### 3. 图表渲染性能
**问题**：大量数据点导致图表卡顿  
**解决**：数据采样、虚拟渲染、按需加载

### 4. 跨域问题
**问题**：前后端分离导致跨域请求被拦截  
**解决**：配置 CORS，允许指定域名访问

### 5. 文件上传大小限制
**问题**：大文件上传失败  
**解决**：配置 Spring Boot 文件上传大小限制

- Redis 6.0+
- Maven 3.6+

### 后端启动
```bash
cd backend
mvn clean install
mvn spring-boot:run
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```

### 访问地址
- 前端：http://localhost:5173
- 后端：http://localhost:8080

### 默认账号
- 系统管理员：admin / 123456
- 馆长：curator01 / 123456
- 审批员：approver01 / 123456
- 借展人：loaner / 123456

### 注意事项
- 首次运行需要执行数据库初始化脚本（包括museum_tables.sql、create_verification_code_table.sql、notification_system.sql）
- 确保MySQL和Redis服务已启动
- 修改application.yml中的数据库连接配置
- 配置邮件服务（修改application-mail-qq.yml或application-mail-163.yml）
- 配置短信服务（修改application.yml中的阿里云SMS配置，或设置sms.enabled=false禁用）
- 借展人用户需要在数据库中设置phone字段才能正常使用申请借展功能
- 借展人登录时必须选择所属博物馆（前台登录页面）
- 新增借展人用户时必须分配博物馆（后台用户管理）
- 系统预置了10个示例博物馆数据
- 查看日志规范和异常处理规范文档（backend/docs/）
- 前端需要建立WebSocket连接才能接收实时通知
