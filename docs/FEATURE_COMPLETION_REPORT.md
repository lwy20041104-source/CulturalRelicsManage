### 4. 魔法数字

## 四、性能优化建议

### 1. N+1查询问题
- 文物列表查询时可能存在N+1问题
- **建议**: 使用JOIN或批量查询

### 2. 缓存策略
- Redis未充分使用
- **建议**: 缓存分类树、统计数据等热点数据

### 3. AI查询性能
- 全网检索时间较长
- **建议**: 添加超时控制和异步处理

### 4. 深度分页
- 使用LIMIT OFFSET性能差
- **建议**: 使用游标分页或Elasticsearch

## 五、安全增强建议

### 1. Token刷新机制
- 配置了refresh-expiration但未实现刷新接口
- **建议**: 实现Token刷新接口

### 2. 文件上传安全
- 缺少严格的文件类型、大小验证
- **建议**: 前后端双重验证，限制文件类型白名单

### 3. SQL注入风险
- 部分动态SQL拼接可能存在风险
- **建议**: 全部使用预编译语句或MyBatis-Plus
