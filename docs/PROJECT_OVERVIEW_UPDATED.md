# 博物馆文物数字化管理系统 - 完整功能文档（2026年4月27日更新）

## 项目概述

博物馆文物数字化管理系统是一个基于 Spring Boot + Vue 3 的全栈 Web 应用，用于博物馆文物的全生命周期管理。

**项目特色**：
- ✅ **28个核心功能模块**，覆盖文物管理全流程
- ✅ **完善的安全机制**（JWT认证、登录限制、密码重置）
- ✅ **实时通知系统**（WebSocket推送、邮件通知）
- ✅ **主题自定义系统**（6个主题 × 2种模式 = 12种视觉风格）
- ✅ **完整的用户管理体系**（个人信息、员工管理、借展人管理）
- ✅ **数字化档案管理**（档案CRUD、文档管理、PDF/Word导出）
- ✅ **AI智能查询**（馆藏优先+全网检索兜底）
- ✅ **批量操作功能**（批量删除、批量修改状态）
- ✅ **前后台分离**（独立登录、独立界面、独立主题）

---

## 核心功能模块（28个）

### 1. 用户认证与权限管理
- JWT Token认证、BCrypt密码加密
- 4种角色（ADMIN/CURATOR/APPROVER/LOANER）
- 基于角色的访问控制（RBAC）

### 2-15. 文物管理相关
- 文物信息管理、文物二维码、文物详情展示
- 文物分类管理、借展管理、修复管理
- 维护记录管理、修复专家管理
- 数据分析与报表、智能仪表盘
- AI智能查询、批量操作
- 图片代理服务、操作日志管理

### 16. 博物馆管理
- 博物馆信息管理（CRUD）
- 用户-博物馆关联管理
- 前台登录博物馆选择

### 17. 前台借展人端
- 独立登录界面（棕色复古风）
- 文物浏览、申请借展、AI查询

### 18. 未登录访客页面 ✨ (新增)
- 独立访客页面（PublicGuestView.vue）
- 古典棕色背景、文物展示
- 登录引导、功能介绍
- 完整国际化支持

### 19-28. 安全与管理功能
- 密码重置功能、登录安全增强
- 数字化档案管理、消息通知系统
- 异常处理与日志规范
- 个人信息管理、员工管理、借展人管理
- 主题自定义系统、暗黑模式

## 项目运行

### 环境要求
- JDK 8+、Node.js 16+
- MySQL 8.0、Redis 6.0+

### 启动命令
```bash
# 后端
cd backend
mvn spring-boot:run

# 前端
cd frontend
npm run dev
```
