# 文物图片上传功能 - 快速开始

## 🎉 新功能介绍

文物管理系统现在支持**两种方式**添加文物图片：

### 方式一：上传图片文件 📤
直接从本地选择图片文件上传到服务器

### 方式二：输入图片URL 🔗
直接填写图片的网络地址

---

## 🚀 快速使用

### 1. 上传图片文件

```
1. 点击"新增文物"
2. 填写文物信息
3. 在"图片"字段，保持"上传图片"模式
4. 拖拽或点击上传图片
5. 点击"确认"保存
```

**效果**：图片保存在服务器，访问稳定

---

### 2. 输入图片URL

```
1. 点击"新增文物"
2. 填写文物信息
3. 在"图片"字段，点击"输入URL"
4. 粘贴图片URL地址
5. 点击"确认"保存
```

**效果**：直接使用网络图片，节省空间

---

## 💡 使用建议

### 什么时候用"上传图片"？
- ✅ 图片在本地电脑上
- ✅ 需要长期稳定访问
- ✅ 图片是独家资源

### 什么时候用"输入URL"？
- ✅ 图片已经在网上
- ✅ 图片来自可信的CDN
- ✅ 需要节省服务器空间

## 🎯 示例

### 示例1：上传本地图片

```
文物名称：青花瓷瓶
年代：明朝
材质：陶瓷
图片：[上传本地图片 vase.jpg]
```

**结果**：图片保存为 `/uploads/relics/20240101_123456_vase.jpg`

---

### 示例2：使用网络图片

```
文物名称：青铜鼎
年代：商朝
材质：青铜
图片URL：https://example.com/images/bronze-ding.jpg
```

**结果**：图片路径保存为完整URL

---

## ⚠️ 注意事项

1. **文件大小**：上传的图片不能超过 5MB
2. **文件格式**：只支持图片格式（JPG/PNG/GIF等）
3. **URL格式**：必须以 http:// 或 https:// 开头
4. **编辑限制**：编辑时暂不支持更换已上传的图片（可切换到URL模式）
5. **网络图片**：确保URL可以公开访问

## 🔧 技术细节

### 前端实现
- **文件**：`frontend/src/views/RelicsView.vue`
- **组件**：Element Plus Upload、Input
- **验证**：文件大小、格式、URL格式

### 后端实现
- **接口1**：`POST /api/relics` - 支持URL
- **接口2**：`POST /api/relics/with-image` - 支持文件上传
- **存储**：`uploads/relics/` 目录

### 数据库
- **字段**：`image_path` (VARCHAR)
- **URL方式**：存储完整URL
- **上传方式**：存储相对路径

## 🆘 常见问题

### Q: 可以同时使用两种方式吗？
**A**: 不可以，每次只能选择一种方式。

### Q: 编辑时可以更换图片吗？
**A**: URL模式可以随时修改，上传模式建议切换到URL模式。

### Q: 图片加载失败怎么办？
**A**: 检查URL是否正确，或切换到上传模式。

### Q: 上传的图片在哪里？
**A**: 保存在服务器的 `uploads/relics/` 目录。
