# 文物二维码功能测试指南

## 测试环境准备

### 1. 启动后端服务
```bash
cd backend
mvn spring-boot:run
```
确保后端服务运行在 `http://localhost:8080`

### 2. 启动前端服务
```bash
cd frontend
npm run dev
```
确保前端服务运行在 `http://localhost:5173`

### 3. 登录系统
- 使用管理员账号登录后台管理系统
- 进入"文物管理"页面

## 功能测试步骤

### 测试1：生成二维码

#### 步骤
1. 在文物列表中选择任意一个文物
2. 点击操作列的"二维码"按钮
3. 观察二维码对话框是否正常弹出
4. 检查二维码图片是否正常显示
5. 检查文物名称和编号是否正确显示

#### 预期结果
- ✅ 对话框正常弹出
- ✅ 二维码图片清晰显示（200x200像素）
- ✅ 文物名称和编号正确
- ✅ 提示信息"扫描二维码可查看文物基本信息"显示

#### 可能的问题
- ❌ 二维码生成失败：检查后端ZXing库是否正确引入
- ❌ 图片不显示：检查Base64格式是否正确
- ❌ 对话框不弹出：检查前端API调用是否成功

## 常见问题排查

### 问题1：二维码生成失败
**现象**：点击"二维码"按钮后，提示"生成二维码失败"

**排查步骤**：
1. 检查浏览器控制台是否有错误
2. 检查后端日志是否有异常
3. 确认ZXing库是否正确引入
4. 检查文物ID是否存在

**解决方案**：
- 重启后端服务
- 检查pom.xml中的依赖
- 查看QRCodeUtil类是否正确

### 问题2：扫描后无法访问
**现象**：扫描二维码后，页面无法打开

**排查步骤**：
1. 检查二维码中的URL是否正确
2. 确认前端服务是否运行
3. 检查网络连接
4. 确认路由配置是否正确

**解决方案**：
- 使用公网IP或域名
- 配置正确的baseUrl
- 检查防火墙设置

### 问题3：文物信息不显示
**现象**：扫描页面打开，但文物信息为空

**排查步骤**：
1. 检查API接口是否正常
2. 确认文物ID是否正确
3. 查看浏览器控制台错误
4. 检查后端日志

**解决方案**：
- 确认getRelicByIdApi接口正常
- 检查文物是否被删除
- 重新生成二维码
