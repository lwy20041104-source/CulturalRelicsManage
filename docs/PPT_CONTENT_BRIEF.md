# 博物馆文物数字化管理系统 PPT 简要内容

## 一、研究背景及意义

### 研究背景
- **文物管理现状**：我国文物超1亿件，传统纸质管理效率低、检索难
- **数字化需求**：国家推动"互联网+中华文明"，疫情加速线上展览需求
- **技术机遇**：云计算、AI、3D建模等技术成熟，为数字化提供支撑

### 研究意义
- **提升效率**：实现文物信息数字化存储，提高管理效率
- **规范流程**：规范借展、修复等业务流程，减少人为错误
- **文化传承**：促进文化遗产保护，提升公共文化服务能力

---

## 二、系统设计

### 技术架构
- **前后端分离**：Vue 3 + Spring Boot
- **数据存储**：MySQL（业务数据）+ Redis（缓存）
- **安全认证**：JWT + Spring Security
- **三层架构**：Controller → Service → Mapper

### 功能模块（27个核心模块）

**用户管理（5个）**
- 认证权限、个人信息、员工管理、借展人管理、登录安全

**文物管理（5个）**
- 文物CRUD、3D展示、二维码标签、详情增强、分类管理

**业务流程（4个）**
- 借展管理、修复管理、维护记录、专家管理

**数据分析（2个）**
- 统计报表、智能仪表盘

**智能功能（1个）**
- AI智能查询（馆藏+全网检索）

**档案管理（1个）**
- 数字化档案（文档管理、PDF/Word导出）

**通知日志（2个）**
- 实时通知（WebSocket）、操作日志

**系统管理（4个）**
- 博物馆管理、批量操作、图片代理、密码重置

**体验优化（2个）**
- 主题系统（6个主题）、暗黑模式（12种风格）

**技术规范（1个）**
- 异常处理与日志规范

### 核心技术亮点

**1. 3D文物展示**
- Three.js渲染引擎，支持GLTF/GLB/OBJ格式
- 交互控制（旋转、缩放、平移）+ 光照系统

**2. 实时通知系统**
- WebSocket + STOMP协议
- 用户私有队列，精准推送

**3. 主题系统**
- 6个主题 × 2种模式 = 12种视觉风格
- 全局CSS变量，动态切换

**4. AI智能查询**
- 双层检索：馆藏优先 + 全网兜底
- 7层图片提取策略

**5. 安全机制**
- BCrypt加密 + JWT认证
- 登录限制（5次失败锁定30分钟）
- 敏感信息脱敏
