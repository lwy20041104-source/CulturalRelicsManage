# 数据备份系统使用指南

## 概述

文物管理系统的数据备份功能提供了完整的数据库备份和恢复能力，确保数据安全和业务连续性。

## 核心特性

### 1. 备份方式
- **手动备份**：管理员可以随时创建备份
- **自动备份**：支持定时自动备份（通过配置）
- **加密备份**：支持AES加密保护敏感数据

### 2. 技术实现
系统采用 **Java 原生方式** 进行备份和恢复，具有以下优势：
- ✅ 不依赖外部命令（mysqldump/mysql）
- ✅ 跨平台兼容（Windows/Linux/Mac）
- ✅ 更可靠和稳定
- ✅ 详细的日志记录

### 3. 备份内容
- 完整的数据库结构（所有表的CREATE TABLE语句）
- 完整的数据记录（所有表的INSERT语句）
- 支持所有数据类型（字符串、数字、日期、二进制等）

## 使用方法

### 一、创建备份

#### 1. 进入备份管理界面
- 登录系统（需要管理员权限）
- 点击左侧菜单 "系统管理" → "数据备份"

#### 2. 创建新备份
1. 点击 "创建备份" 按钮
2. 填写备份信息：
   - **备份名称**：为备份起一个有意义的名称（必填）
   - **备份描述**：说明备份的目的或内容（可选）
   - **是否加密**：选择是否对备份文件进行AES加密
3. 点击 "确定" 开始备份

#### 3. 备份过程
- 备份状态会显示为 "处理中"
- 系统在后台异步执行备份
- 备份完成后状态会自动更新为 "成功" 或 "失败"
- 可以看到备份文件大小和创建时间

#### 4. 备份文件
- 备份文件存储在 `backend/backups/` 目录
- 文件命名格式：`backup_YYYYMMDD_HHmmss.sql`
- 例如：`backup_20260509_145118.sql`

### 二、查看备份列表

#### 1. 筛选功能
- **备份类型**：手动备份 / 自动备份
- **备份状态**：成功 / 失败 / 处理中
- **搜索**：按备份名称搜索

#### 2. 显示信息
每个备份记录显示以下信息：
- 备份名称
- 备份类型（手动/自动）
- 备份状态（成功/失败/处理中）
- 文件大小（自动格式化为 KB/MB/GB）
- 是否加密
- 创建人（显示真实姓名）
- 创建时间
- 备份描述

### 三、下载备份

1. 在备份列表中找到要下载的备份
2. 点击 "下载" 按钮
3. 浏览器会自动下载备份文件
4. 如果备份已加密，下载时会自动解密

### 四、恢复数据库

⚠️ **重要提示**：恢复操作会覆盖当前数据库，请谨慎操作！

#### 1. 恢复步骤
1. 在备份列表中找到要恢复的备份
2. 点击 "恢复" 按钮
3. 系统会弹出确认对话框
4. 确认后开始恢复

#### 2. 恢复过程
- 系统会创建恢复记录
- 在后台异步执行恢复
- 恢复完成后会显示成功或失败状态
- 如果备份已加密，会自动解密后恢复

#### 3. 注意事项
- 只能恢复状态为 "成功" 的备份
- 恢复过程中不要关闭系统
- 恢复完成后建议重新登录系统

### 五、删除备份

1. 在备份列表中找到要删除的备份
2. 点击 "删除" 按钮
3. 确认删除
4. 系统会同时删除数据库记录和备份文件

### 六、备份配置

#### 1. 进入配置界面
- 在备份管理界面点击 "备份配置" 按钮

#### 2. 配置项说明
- **配置名称**：配置的名称
- **是否启用**：启用或禁用自动备份
- **备份频率**：每日/每周/每月
- **备份时间**：自动备份的执行时间（格式：HH:mm）
- **保留天数**：备份文件保留的天数
- **最大备份数**：最多保留的备份文件数量
- **是否加密**：自动备份是否加密
- **通知启用**：备份完成后是否发送通知

#### 3. 清理过期备份
- 点击 "清理过期备份" 按钮
- 系统会根据配置的保留天数自动删除过期备份

## 技术架构

### 1. 后端实现

#### 核心类
- **BackupController**：备份管理控制器
  - 处理HTTP请求
  - 权限控制（需要ADMIN角色）
  - 审计日志记录

- **BackupService**：备份服务
  - 创建备份记录
  - 异步执行备份/恢复
  - 文件加密/解密
  - 配置管理

- **DatabaseBackupUtil**：数据库备份工具
  - Java原生方式备份
  - 读取表结构和数据
  - 生成SQL文件
  - 执行SQL恢复

#### 数据库表
- **sys_backup**：备份记录表
  - 存储备份的元数据
  - 记录备份状态和文件信息

- **sys_backup_config**：备份配置表
  - 存储自动备份配置

- **sys_restore**：恢复记录表
  - 记录恢复操作的历史

### 2. 备份流程

```
1. 用户创建备份
   ↓
2. 创建备份记录（状态：处理中）
   ↓
3. 异步执行备份
   ├─ 创建备份目录
   ├─ 连接数据库
   ├─ 获取所有表
   ├─ 逐表备份（结构+数据）
   ├─ 生成SQL文件
   └─ 可选：加密文件
   ↓
4. 更新备份记录（状态：成功/失败）
```

### 3. 恢复流程

```
1. 用户选择备份恢复
   ↓
2. 创建恢复记录（状态：处理中）
   ↓
3. 异步执行恢复
   ├─ 检查备份文件
   ├─ 可选：解密文件
   ├─ 连接数据库
   ├─ 读取SQL文件
   ├─ 逐条执行SQL语句
   └─ 记录执行结果
   ↓
4. 更新恢复记录（状态：成功/失败）
```

### 4. 加密机制

- **算法**：AES（高级加密标准）
- **密钥**：配置在 `BackupService.AES_KEY`
- **流程**：
  1. 备份完成后，读取SQL文件
  2. 使用AES加密整个文件内容
  3. 写回原文件
  4. 恢复时自动解密到临时文件

## 常见问题

### Q1: 备份一直显示"处理中"？
**A**: 检查后端日志，可能原因：
- 数据库连接失败
- 磁盘空间不足
- 权限问题

### Q2: 备份文件大小为0B？
**A**: 已修复，现在使用Java原生方式，不依赖外部命令。

### Q3: 恢复失败怎么办？
**A**: 
1. 检查备份文件是否完整
2. 查看后端日志的详细错误信息
3. 确认数据库连接正常
4. 尝试使用其他备份文件

### Q4: 可以恢复到不同的数据库吗？
**A**: 可以，但需要修改备份文件中的数据库名称。

### Q5: 备份文件可以手动编辑吗？
**A**: 
- 未加密的备份：可以用文本编辑器打开和编辑
- 已加密的备份：需要先解密才能编辑
