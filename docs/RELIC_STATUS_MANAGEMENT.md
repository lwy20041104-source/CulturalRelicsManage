# 文物状态管理规则

## 概述

本文档定义了文物管理系统中文物状态的管理规则，确保文物在借展、修复等业务流程中的状态一致性和业务逻辑正确性。

## 文物状态定义

文物在系统中有以下几种状态：

| 状态 | 说明 | 可进行的操作 |
|------|------|------------|
| 在库 | 文物正常存放在库房中 | 申请借展、申请修复、查看、编辑 |
| 借展中 | 文物正在被借展 | 查看、归还 |
| 修复中 | 文物正在修复 | 查看、更新修复进度、完成修复 |
| 展出中 | 文物正在展出 | 查看 |
| 遗失 | 文物遗失 | 查看 |
| 损毁 | 文物损毁 | 查看、申请修复 |

## 核心业务规则

### 规则1：借展申请限制

**规则**：只有状态为"在库"的文物才能申请借展

**原因**：
- 借展中的文物不能重复借展
- 修复中的文物不能借展（需要先完成修复）
- 展出中的文物不能借展
- 遗失或损毁的文物不能借展

**实现位置**：`LoanRecordServiceImpl.save()`

**错误提示**：
```
只有在库状态的文物才能申请借展，当前状态：{实际状态}
```

**示例**：
```java
// 正确：在库状态可以申请借展
文物状态：在库 → 申请借展 → 成功

// 错误：修复中不能申请借展
文物状态：修复中 → 申请借展 → 失败（提示：只有在库状态的文物才能申请借展，当前状态：修复中）

// 错误：借展中不能重复借展
文物状态：借展中 → 申请借展 → 失败（提示：只有在库状态的文物才能申请借展，当前状态：借展中）
```

### 规则2：修复申请限制

**规则**：只有状态为"在库"的文物才能申请修复

**原因**：
- 借展中的文物不能申请修复（需要先归还）
- 修复中的文物不能重复申请修复
- 展出中的文物不能申请修复

**实现位置**：`RepairRecordServiceImpl.applyRepair()`

**错误提示**：
```
只有在库状态的文物才能申请修复，当前状态：{实际状态}
```

**示例**：
```java
// 正确：在库状态可以申请修复
文物状态：在库 → 申请修复 → 成功

// 错误：借展中不能申请修复
文物状态：借展中 → 申请修复 → 失败（提示：只有在库状态的文物才能申请修复，当前状态：借展中）

// 错误：修复中不能重复申请修复
文物状态：修复中 → 申请修复 → 失败（提示：只有在库状态的文物才能申请修复，当前状态：修复中）
```

### 规则3：文物状态自动更新

**规则**：文物状态随业务流程自动更新

#### 3.1 借展流程中的状态变化

| 操作 | 状态变化 | 实现位置 |
|------|---------|---------|
| 申请借展 | 在库 → 在库（待审批） | `LoanRecordServiceImpl.save()` |
| 审批通过 | 在库 → 借展中 | `LoanRecordServiceImpl.approveLoan()` |
| 审批拒绝 | 在库 → 在库 | `LoanRecordServiceImpl.approveLoan()` |
| 归还 | 借展中 → 在库 | `LoanRecordServiceImpl.returnLoan()` |

#### 3.2 修复流程中的状态变化

| 操作 | 状态变化 | 实现位置 |
|------|---------|---------|
| 申请修复 | 在库 → 在库（待审批） | `RepairRecordServiceImpl.applyRepair()` |
| 审批通过 | 在库 → 在库（待修复） | `RepairRecordServiceImpl.approveRepair()` |
| 审批拒绝 | 在库 → 在库 | `RepairRecordServiceImpl.approveRepair()` |
| 开始修复 | 在库 → 修复中 | `RepairRecordServiceImpl.startRepair()` |
| 完成修复 | 修复中 → 在库 | `RepairRecordServiceImpl.completeRepair()` |

## 完整业务流程

### 借展流程

```
1. 申请借展
   - 检查：文物状态必须为"在库"
   - 创建：借展记录（状态：待审批）
   - 文物状态：保持"在库"

2. 审批借展
   - 审批通过：
     * 借展记录状态：待审批 → 借出中
     * 文物状态：在库 → 借展中
   - 审批拒绝：
     * 借展记录状态：待审批 → 已驳回
     * 文物状态：保持"在库"

3. 归还文物
   - 借展记录状态：借出中 → 已归还
   - 文物状态：借展中 → 在库
```

### 修复流程

```
1. 申请修复
   - 检查：文物状态必须为"在库"
   - 创建：修复记录（状态：待审批）
   - 文物状态：保持"在库"

2. 审批修复
   - 审批通过：
     * 修复记录状态：待审批 → 待修复
     * 文物状态：保持"在库"
   - 审批拒绝：
     * 修复记录状态：待审批 → 已拒绝
     * 文物状态：保持"在库"

3. 开始修复
   - 修复记录状态：待修复 → 修复中
   - 文物状态：在库 → 修复中

4. 完成修复
   - 修复记录状态：修复中 → 修复完成
   - 文物状态：修复中 → 在库
```

## 状态冲突场景

### 场景1：文物正在借展，尝试申请修复

```
文物状态：借展中
操作：申请修复
结果：失败
错误提示：只有在库状态的文物才能申请修复，当前状态：借展中
解决方案：等待文物归还后再申请修复
```

### 场景2：文物正在修复，尝试申请借展

```
文物状态：修复中
操作：申请借展
结果：失败
错误提示：只有在库状态的文物才能申请借展，当前状态：修复中
解决方案：等待修复完成后再申请借展
```

### 场景3：文物正在借展，尝试再次借展

```
文物状态：借展中
操作：申请借展
结果：失败
错误提示：只有在库状态的文物才能申请借展，当前状态：借展中
解决方案：等待文物归还后再申请借展
```

### 场景4：文物正在修复，尝试再次修复

```
文物状态：修复中
操作：申请修复
结果：失败
错误提示：只有在库状态的文物才能申请修复，当前状态：修复中
解决方案：等待当前修复完成后再申请新的修复
```

## 数据库设计

### 文物表（cultural_relic）

```sql
CREATE TABLE cultural_relic (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    relic_code VARCHAR(50) UNIQUE NOT NULL COMMENT '文物编号',
    relic_name VARCHAR(200) NOT NULL COMMENT '文物名称',
    status VARCHAR(20) DEFAULT '在库' COMMENT '状态：在库、借展中、修复中、展出中、遗失、损毁',
    -- 其他字段...
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 借展记录表（loan_record）

```sql
CREATE TABLE loan_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    relic_id BIGINT NOT NULL COMMENT '文物ID',
    status VARCHAR(20) DEFAULT '待审批' COMMENT '状态：待审批、借出中、已归还、已驳回',
    -- 其他字段...
    FOREIGN KEY (relic_id) REFERENCES cultural_relic(id)
);
```

### 修复记录表（repair_record）

```sql
CREATE TABLE repair_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    relic_id BIGINT NOT NULL COMMENT '文物ID',
    status VARCHAR(20) DEFAULT '待审批' COMMENT '状态：待审批、待修复、修复中、修复完成、已拒绝',
    -- 其他字段...
    FOREIGN KEY (relic_id) REFERENCES cultural_relic(id)
);
```

## 前端处理

### 1. 申请借展时的前端验证

```javascript
// 在申请借展前检查文物状态
if (relic.status !== '在库') {
  ElMessage.error(`只有在库状态的文物才能申请借展，当前状态：${relic.status}`)
  return
}
```

### 2. 申请修复时的前端验证

```javascript
// 在申请修复前检查文物状态
if (relic.status !== '在库') {
  ElMessage.error(`只有在库状态的文物才能申请修复，当前状态：${relic.status}`)
  return
}
```

### 3. 文物列表中的操作按钮控制

```vue
<template>
  <!-- 借展按钮：只有在库状态才显示 -->
  <el-button 
    v-if="row.status === '在库'" 
    @click="handleLoan(row)"
  >
    申请借展
  </el-button>
  
  <!-- 修复按钮：只有在库状态才显示 -->
  <el-button 
    v-if="row.status === '在库'" 
    @click="handleRepair(row)"
  >
    申请修复
  </el-button>
</template>
```

## 实现文件清单

### 后端文件

1. **LoanRecordServiceImpl.java**
   - `save()` - 添加文物状态检查
   - `approveLoan()` - 审批通过时更新文物状态
   - `returnLoan()` - 归还时更新文物状态

2. **RepairRecordServiceImpl.java**
   - `applyRepair()` - 添加文物状态检查
   - `startRepair()` - 开始修复时更新文物状态
   - `completeRepair()` - 完成修复时更新文物状态

3. **RepairRecordMapper.java**
   - `selectRelicById()` - 查询文物信息
   - `updateRelicStatus()` - 更新文物状态

4. **RepairRecordMapper.xml**
   - 添加查询和更新文物状态的SQL

### 前端文件（建议修改）

1. **RelicsView.vue** - 文物管理页面
   - 根据文物状态控制操作按钮显示

2. **LoansView.vue** - 借展管理页面
   - 申请借展前验证文物状态

3. **RepairsView.vue** - 修复管理页面
   - 申请修复前验证文物状态
