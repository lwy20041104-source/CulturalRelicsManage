# 登录安全机制文档

## 概述

本系统实现了完善的登录安全机制，防止暴力破解攻击，保护用户账户安全。

## 核心功能

### 1. 登录失败次数限制

**规则**:
- 最大失败次数：**3次**
- 达到最大次数后：账户自动锁定
- 锁定时长：**30分钟**

**实现逻辑**:
1. 用户输入错误密码时，系统记录失败次数
2. 每次失败后显示剩余尝试次数
3. 第3次失败后，账户立即锁定30分钟

### 2. 账户锁定机制

**锁定条件**:
- 连续3次密码错误
- 管理员手动锁定

**锁定效果**:
- 无法登录系统
- 显示锁定提示信息
- 记录锁定时间和IP地址

**解锁方式**:
1. **自动解锁**: 锁定30分钟后自动解除
2. **管理员解锁**: 管理员可手动解锁账户

### 3. 安全提示信息

**登录失败提示**:
- 第1次失败：`密码错误，还剩 2 次尝试机会`
- 第2次失败：`密码错误，还剩 1 次尝试机会`
- 第3次失败：`密码错误次数过多，账户已被锁定 30 分钟`

**账户锁定提示**:
- `账户已被锁定，请在 30 分钟后重试或联系管理员`

## 数据库设计

### sys_user表新增字段

```sql
ALTER TABLE sys_user 
ADD COLUMN login_failed_count INT DEFAULT 0 COMMENT '登录失败次数',
ADD COLUMN account_locked TINYINT(1) DEFAULT 0 COMMENT '账户是否锁定：0-未锁定，1-已锁定',
ADD COLUMN locked_time DATETIME NULL COMMENT '账户锁定时间',
ADD COLUMN last_login_time DATETIME NULL COMMENT '最后登录时间',
ADD COLUMN last_login_ip VARCHAR(50) NULL COMMENT '最后登录IP';
```

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| login_failed_count | INT | 登录失败次数，成功登录后重置为0 |
| account_locked | TINYINT(1) | 账户锁定状态：0-未锁定，1-已锁定 |
| locked_time | DATETIME | 账户锁定时间，用于计算自动解锁时间 |
| last_login_time | DATETIME | 最后一次成功登录时间 |
| last_login_ip | VARCHAR(50) | 最后一次登录的IP地址 |

## 核心服务

### LoginSecurityService

登录安全服务接口，提供以下功能：

#### 常量配置

```java
int MAX_FAILED_ATTEMPTS = 3;           // 最大失败次数
int LOCK_DURATION_MINUTES = 30;        // 锁定时长（分钟）
```

#### 核心方法

1. **isAccountLocked(String username)**
   - 检查账户是否被锁定
   - 自动检查锁定是否过期

2. **recordLoginFailure(String username, String ipAddress)**
   - 记录登录失败
   - 增加失败次数
   - 达到上限时自动锁定

3. **lockAccount(String username)**
   - 手动锁定账户
   - 记录锁定时间

4. **resetLoginFailures(String username, String ipAddress)**
   - 登录成功后重置失败次数
   - 更新最后登录时间和IP

5. **unlockAccount(String username)**
   - 解锁账户
   - 重置失败次数

6. **getRemainingAttempts(String username)**
   - 获取剩余尝试次数

7. **checkAndAutoUnlock(String username)**
   - 检查并自动解锁过期的锁定

## 登录流程

### 正常登录流程

```
1. 用户输入用户名和密码
   ↓
2. 检查账户是否被锁定
   ↓
3. 验证用户名是否存在
   ↓
4. 验证密码是否正确
   ↓
5. 验证角色是否匹配
   ↓
6. 重置失败次数
   ↓
7. 更新最后登录时间和IP
   ↓
8. 生成JWT Token
   ↓
9. 返回登录成功
```

### 失败处理流程

```
1. 密码验证失败
   ↓
2. 记录失败次数 +1
   ↓
3. 检查失败次数
   ↓
4. 如果 < 3次
   │  └─> 返回剩余尝试次数
   │
5. 如果 = 3次
   └─> 锁定账户
       └─> 记录锁定时间
           └─> 返回锁定提示
```

## API接口

### 1. 登录接口

**接口**: `POST /api/auth/login`

**请求参数**:
```json
{
  "username": "admin",
  "password": "123456",
  "roleCode": "ADMIN"
}
```

**成功响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "admin",
    "realName": "系统管理员",
    "userId": 1,
    "phone": "13800138000"
  }
}
```

**失败响应（密码错误）**:
```json
{
  "code": 500,
  "message": "密码错误，还剩 2 次尝试机会"
}
```

**失败响应（账户锁定）**:
```json
{
  "code": 500,
  "message": "账户已被锁定，请在 30 分钟后重试或联系管理员"
}
```

### 2. 解锁账户接口

**接口**: `POST /api/users/unlock-by-username`

**请求参数**:
```
username=testuser
```

**成功响应**:
```json
{
  "code": 200,
  "message": "账户解锁成功",
  "data": true
}
```

**权限要求**: 仅管理员可操作

## 安全日志

### 登录日志记录

系统自动记录以下登录相关日志：

1. **登录成功日志**
   - 操作人：用户真实姓名
   - 操作类型：登录
   - 操作模块：系统认证
   - 操作内容：用户登录
   - 操作结果：成功
   - IP地址：客户端IP

2. **登录失败日志**
   - 操作人：用户名
   - 操作类型：登录
   - 操作模块：系统认证
   - 操作内容：用户登录
   - 操作结果：失败
   - IP地址：客户端IP

3. **账户锁定日志**
   - 记录在应用日志中
   - 包含用户名、失败次数、IP地址

### 日志查询

管理员可通过"操作日志"模块查询登录相关日志：
- 筛选操作类型：登录
- 筛选操作模块：系统认证
- 查看失败原因和IP地址

## 配置说明

### 修改最大失败次数

编辑 `LoginSecurityService.java`:
```java
int MAX_FAILED_ATTEMPTS = 5;  // 改为5次
```

### 修改锁定时长

编辑 `LoginSecurityService.java`:
```java
int LOCK_DURATION_MINUTES = 60;  // 改为60分钟
```

### 禁用自动解锁

如需禁用自动解锁，修改 `LoginSecurityServiceImpl.java` 的 `isAccountLocked` 方法，移除自动解锁逻辑。

## 故障排查

### 问题1：账户无法自动解锁

**原因**:
- 服务器时间不准确
- 锁定时间字段为空

**解决方案**:
1. 检查服务器时间
2. 检查数据库locked_time字段
3. 手动执行解锁操作

### 问题2：失败次数未重置

**原因**:
- 登录成功后未调用重置方法
- 数据库更新失败

**解决方案**:
1. 检查登录成功后的代码逻辑
2. 检查数据库连接
3. 查看应用日志

### 问题3：所有用户都无法登录

**原因**:
- 数据库字段未添加
- 服务启动失败

**解决方案**:
1. 执行SQL脚本添加字段
2. 重启应用服务
3. 检查应用启动日志
