# 文物二维码功能总结

## 📋 功能概述

文物二维码功能允许为每个文物生成唯一的二维码标签，扫描后可以查看文物的基本信息。

---

## ✅ 已实现的功能

### 1. 二维码生成
- ✅ 单个文物生成二维码
- ✅ 批量生成二维码
- ✅ 带标签的二维码（包含文物编号和名称）
- ✅ Base64格式输出
- ✅ 200x200像素标准尺寸

### 2. 二维码内容
- ✅ 包含文物ID的URL
- ✅ 自动适配访问地址（localhost或局域网IP）
- ✅ 格式：`http://域名/qrcode/{文物ID}`

### 3. 扫描后展示
- ✅ 专门的扫描页面（QRCodeScanView）
- ✅ 显示文物完整信息
- ✅ 显示文物图片（支持预览）
- ✅ 无需登录即可访问
- ✅ 美观的UI设计

### 4. 下载和打印
- ✅ 下载二维码图片
- ✅ 文件名自动包含文物编号和名称
- ✅ 支持打印预览

### 5. 公开访问
- ✅ 后端配置允许公开访问
- ✅ 无需token验证
- ✅ 安全的只读访问

## 🌐 网络访问说明

### ❌ 当前问题

**如果项目只在本地运行（localhost），其他设备无法扫描二维码访问。**

原因：
- `localhost` 只能在本机访问
- 其他设备（手机、平板）无法通过 `localhost` 访问你的电脑
- 二维码中的URL是 `http://localhost:5173/qrcode/{id}`

### ✅ 解决方案

#### 方案1：局域网IP访问（推荐测试用）

**适用场景**：
- 同一WiFi网络下的设备
- 内网演示、测试
- 快速验证功能

**配置步骤**：
1. 获取电脑IP地址（如 `192.168.1.100`）
2. 使用IP地址访问：`http://192.168.1.100:5173`
3. 生成的二维码会自动包含IP地址
4. 同一WiFi下的设备可以扫描访问

**优点**：
- ✅ 配置简单，无需修改代码
- ✅ 速度快，无延迟
- ✅ 适合内部测试

**缺点**：
- ❌ 只能同一WiFi访问
- ❌ IP地址可能变化
- ❌ 需要配置防火墙

#### 方案2：内网穿透（推荐演示用）

**适用场景**：
- 远程演示
- 临时分享给他人
- 不在同一网络的设备

**工具推荐**：
- ngrok（国外）
- 花生壳（国内）
- frp（自建）

**优点**：
- ✅ 外网可访问
- ✅ 无需服务器
- ✅ 配置简单

**缺点**：
- ❌ 依赖第三方服务
- ❌ 速度可能较慢
- ❌ 免费版有限制

#### 方案3：服务器部署（推荐生产用）

**适用场景**：
- 正式使用
- 长期运行
- 多人访问

**部署内容**：
- 云服务器（阿里云、腾讯云等）
- 域名和SSL证书
- Nginx反向代理

**优点**：
- ✅ 稳定可靠
- ✅ 性能好
- ✅ 任何设备可访问

**缺点**：
- ❌ 需要服务器成本
- ❌ 配置较复杂
- ❌ 需要运维知识

## 🚀 快速开始（局域网访问）

### 1. 获取IP地址

**Windows**：
```bash
ipconfig
# 找到 IPv4 地址，如：192.168.1.100
```

**Mac/Linux**：
```bash
ifconfig | grep "inet "
# 找到类似 192.168.1.100 的地址
```

### 2. 启动服务

```bash
# 后端
cd backend
mvn spring-boot:run

# 前端
cd frontend
npm run dev
```

### 3. 访问测试

在手机浏览器输入：
```
http://192.168.1.100:5173/public-relics
```

### 4. 生成二维码

1. 点击任意文物查看详情
2. 查看二维码（应该包含你的IP地址）
3. 用另一台手机扫描
4. 应该能看到文物信息

## ⚠️ 注意事项

### 开发环境

1. **localhost限制**
   - 只能本机访问
   - 其他设备无法扫描
   - 需要使用IP地址

2. **防火墙设置**
   - Windows防火墙可能阻止访问
   - 需要添加入站规则
   - 允许端口5173和8080

3. **IP地址变化**
   - 路由器可能重新分配IP
   - 建议设置静态IP
   - 或使用内网穿透

### 生产环境

1. **服务器部署**
   - 使用固定域名
   - 配置SSL证书
   - 使用HTTPS协议

2. **安全考虑**
   - 只读访问
   - 不暴露敏感信息
   - 限制访问频率

3. **性能优化**
   - 二维码缓存
   - CDN加速
   - 图片压缩
