## 核心业务流程架构

### 1. 借展业务流程

```
┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│ 借展申请  │ ───> │ 审批流程  │ ───> │ 借出管理  │ ───> │ 归还管理  │
└──────────┘      └──────────┘      └──────────┘      └──────────┘
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
  填写申请          审批员审批         文物状态更新        归还确认
  选择文物          通过/驳回          借展中              状态恢复
  设置日期          发送通知           逾期检查            完成记录
```

### 2. 修复业务流程

```
┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│ 修复申请  │ ───> │ 审批流程  │ ───> │ 修复进行  │ ───> │ 修复完成  │
└──────────┘      └──────────┘      └──────────┘      └──────────┘
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
  填写原因          保管员审批         分配专家            验收确认
  选择专家          通过/拒绝          进度跟踪            状态恢复
  设置优先级        发送通知           修复中              完成记录
```

### 3. AI查询流程

```
┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│ 用户输入  │ ───> │ 关键词处理│ ───> │ 双层检索  │ ───> │ 结果展示  │
└──────────┘      └──────────┘      └──────────┘      └──────────┘
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
  输入问题          分词处理           馆藏检索            文物列表
  多轮对话          同义词扩展         全网检索            图片展示
  会话管理          关键词匹配         图片抓取            相关推荐
```

## 说明

以上架构图展示了博物馆文物数字化管理系统的：
1. **整体架构**：前后端分离的三层架构
2. **功能模块**：27个核心模块的组织结构
3. **用户角色**：4种角色的权限分配
4. **业务流程**：借展、修复、AI查询的流程
5. **技术架构**：前后端技术栈的组成
6. **数据库架构**：15张核心表的关系
7. **安全架构**：多层次的安全防护体系
8. **部署架构**：未来的高可用部署方案
9. **监控架构**：未来的监控告警系统

这些架构图可以直接用于PPT制作或系统文档。
