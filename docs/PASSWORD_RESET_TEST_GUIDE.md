# 密码重置功能测试指南

## 测试前准备

### 1. 确保数据库表已创建
运行SQL脚本：
```bash
backend/sql/create_verification_code_table.sql
```

### 2. 确保用户有绑定的邮箱或手机号
检查 `sys_user` 表中的测试用户是否有 `email` 或 `phone` 字段的值。

```sql
-- 更新测试用户的手机号

```

## 测试步骤

### 测试场景1：邮箱验证方式

#### 步骤1：访问忘记密码页面
1. 打开浏览器访问：`http://localhost:5173/forgot-password`
2. 或在登录页面点击"忘记密码？"链接

#### 步骤2：填写表单
1. 输入用户名：`testuser`
2. 选择验证方式：`邮箱验证`
3. 输入邮箱地址：`test@example.com`（必须与数据库中的邮箱匹配）
4. 点击"发送验证码"按钮

#### 步骤3：查看验证码
由于当前是模拟发送，需要在后端日志中查看验证码：
```bash
# 查看后端控制台输出
【模拟发送邮件】收件人：test@example.com，验证码：123456，有效期：15分钟
```

#### 步骤4：重置密码
1. 系统自动跳转到重置密码页面
2. 用户名已自动填充（禁用状态）
3. 输入验证码：`123456`（从日志中获取）
4. 输入新密码：`newpass123`
5. 再次输入新密码：`newpass123`
6. 点击"重置密码"按钮

#### 步骤5：验证结果
1. 看到成功提示："密码重置成功，请使用新密码登录"
2. 自动跳转到登录页面
3. 使用新密码登录：
   - 用户名：`testuser`
   - 密码：`newpass123`
4. 登录成功

### 测试场景3：异常情况测试

#### 3.1 用户不存在
- 输入不存在的用户名
- 预期结果：提示"用户不存在"

#### 3.2 联系方式不匹配
- 输入正确的用户名
- 输入错误的邮箱或手机号（与数据库中不匹配）
- 预期结果：
  - 邮箱：提示"邮箱地址与账户绑定的邮箱不匹配"
  - 手机：提示"手机号与账户绑定的手机号不匹配"

- 在重置密码页面输入错误的验证码

#### 3.4 验证码过期
- 等待15分钟后使用验证码
- 预期结果：提示"验证码无效或已过期"

#### 3.5 验证码重复使用
- 使用同一个验证码重置密码两次

#### 3.7 密码长度不足
- 输入少于6位的密码
- 预期结果：提示"密码长度不能少于6位"

## 数据库验证

### 1. 验证码记录
```sql
-- 查看验证码记录
SELECT * FROM verification_code 
WHERE username = 'testuser' 
ORDER BY create_time DESC;
```

应该看到：
- `code`: 验证码
- `type`: EMAIL 或 PHONE
- `contact`: 邮箱或手机号
- `purpose`: RESET_PASSWORD
- `used`: 0（未使用）或 1（已使用）
- `expire_time`: 过期时间（创建时间 + 15分钟）

### 2. 用户密码更新
```sql
-- 查看用户信息
SELECT username, password, login_failed_count, account_locked, locked_time 
FROM sys_user 
WHERE username = 'testuser';
```

密码重置后应该看到：
- `password`: 新的BCrypt加密密码
- `login_failed_count`: 0
- `account_locked`: 0
- `locked_time`: NULL

### 3. 操作日志
```sql
-- 查看操作日志
SELECT * FROM sys_operation_log 
WHERE operator = 'testuser' 
AND operation_type IN ('忘记密码', '重置密码')
ORDER BY operation_time DESC;
```

应该看到两条记录：
1. 操作类型：忘记密码，操作内容：发送验证码：EMAIL/PHONE
2. 操作类型：重置密码，操作内容：密码重置成功

## API接口测试

### 使用Postman或curl测试

#### 1. 发送验证码
```bash
curl -X POST http://localhost:8080/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "verificationType": "EMAIL",
    "email": "test@example.com"
  }'
```

预期响应：
```json
{
  "code": 200,
  "message": "验证码已发送到您的邮箱：te***@example.com",
  "data": "验证码已发送到您的邮箱：te***@example.com"
}
```

#### 2. 验证验证码
```bash
curl -X POST "http://localhost:8080/auth/verify-code?username=testuser&code=123456"
```

预期响应：
```json
{
  "code": 200,
  "message": "验证码正确",
  "data": true
}
```

#### 3. 重置密码
```bash
curl -X POST http://localhost:8080/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "code": "123456",
    "newPassword": "newpass123",
    "confirmPassword": "newpass123"
  }'
```

预期响应：
```json
{
  "code": 200,
  "message": "密码重置成功，请使用新密码登录",
  "data": true
}
```

## 常见问题

### Q1: 提示"用户不存在"
**A:** 检查数据库中是否有该用户，确认用户名拼写正确。

### Q2: 提示"邮箱地址与账户绑定的邮箱不匹配"
**A:** 检查数据库中该用户的 `email` 字段是否与输入的邮箱一致。

### Q3: 提示"手机号与账户绑定的手机号不匹配"
**A:** 检查数据库中该用户的 `phone` 字段是否与输入的手机号一致。

### Q4: 看不到验证码
**A:** 当前是模拟发送，需要在后端控制台日志中查看验证码。生产环境需要集成实际的邮件或短信服务。

### Q5: 验证码总是提示"无效或已过期"
**A:** 
1. 检查验证码是否输入正确（6位数字）
2. 检查是否超过15分钟有效期
3. 检查验证码是否已经使用过
4. 查看数据库 `verification_code` 表确认记录

### Q6: 密码重置成功但无法登录
**A:** 
1. 确认使用的是新密码
2. 检查数据库中密码是否已更新
3. 检查账户是否被锁定（应该已自动解锁）
