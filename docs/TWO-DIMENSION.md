📱 手机扫描二维码完整步骤指南
🎯 目标
让手机扫描二维码后能看到文物的完整信息、图片、美观页面，无需登录。

方法一：局域网访问（最简单，推荐！）
📋 前提条件
✅ 手机和电脑连接同一个WiFi
✅ 前后端服务正在运行
🚀 详细步骤
第1步：获取电脑的IP地址
Windows系统：

按键盘 Win + R
输入 cmd，按回车
输入 ipconfig，按回车
找到 无线局域网适配器 WLAN 或 以太网适配器
记下 IPv4 地址，例如：192.168.1.100
无线局域网适配器 WLAN:
   IPv4 地址 . . . . . . . . . . . . : 192.168.1.100  ← 这个就是！
Mac系统：

打开终端
输入 ifconfig | grep "inet "
找到类似 192.168.1.100 的地址
第2步：检查防火墙（重要！）
Windows防火墙设置：

方法A：临时关闭（仅测试用）

打开 控制面板
点击 Windows Defender 防火墙
点击左侧 启用或关闭 Windows Defender 防火墙
在 专用网络设置 下选择 关闭 Windows Defender 防火墙
点击 确定
方法B：添加规则（推荐）

打开 控制面板 → Windows Defender 防火墙
点击左侧 高级设置
点击左侧 入站规则
点击右侧 新建规则
选择 端口，点击 下一步
选择 TCP，输入特定本地端口：5173,8080
点击 下一步，选择 允许连接
点击 下一步，全部勾选（域、专用、公用）
点击 下一步，输入名称：文物管理系统
点击 完成
第3步：启动前后端服务
启动后端：

cd backend
mvn spring-boot:run
等待看到 Started CulturalRelicsApplication 字样

启动前端：

cd frontend
npm run dev
等待看到：

➜  Local:   http://localhost:5173/
➜  Network: http://192.168.1.100:5173/  ← 注意这个地址！
第4步：在手机上测试访问
打开手机浏览器（Safari、Chrome等）
输入地址：http://你的IP地址:5173
例如：http://192.168.1.100:5173
如果能看到登录页面，说明配置成功！✅
第5步：访问公开文物页面
在手机浏览器输入：

http://192.168.1.100:5173/public-relics
你应该能看到：

✅ 文物列表（网格布局）
✅ 文物图片
✅ 文物基本信息
第6步：查看二维码
点击任意文物卡片
在详情对话框中，你会看到二维码
重要：检查二维码下方的URL，应该是：
http://192.168.1.100:5173/qrcode/1
而不是 http://localhost:5173/qrcode/1
第7步：扫描二维码
用另一台手机打开相机或扫码APP
扫描电脑屏幕上的二维码
点击弹出的链接
你应该能看到：
✅ 文物名称和编号
✅ 文物图片（可点击放大）
✅ 完整的基本信息（年代、材质、分类、状态、尺寸、重量、来源、描述）
✅ 美观的展示页面
✅ 无需登录
✅ 可以点击"查看详情"或"返回首页"
🎬 完整演示流程
你的电脑（192.168.1.100）
├── 后端运行在 :8080
├── 前端运行在 :5173
└── 连接WiFi：家里的WiFi

手机A（用来生成二维码）

├── 点击文物查看详情

手机B（用来扫描二维码）

├── 打开相机扫描二维码
├── 点击链接
└── 看到文物完整信息 ✅
⚠️ 常见问题和解决方法
问题1：手机无法访问 http://192.168.1.100:5173
可能原因：

手机和电脑不在同一WiFi
防火墙阻止了访问
IP地址输入错误
解决方法：

确认手机和电脑连接同一WiFi
关闭防火墙测试（或添加防火墙规则）
重新检查IP地址是否正确
尝试在电脑浏览器访问 http://192.168.1.100:5173，如果电脑也访问不了，说明配置有问题
问题2：二维码显示 localhost
原因：你在电脑上使用 http://localhost:5173 访问的

解决方法：

❌ 不要用：http://localhost:5173/public-relics
✅ 要用：http://192.168.1.100:5173/public-relics（你的IP）
重新用IP地址访问，生成的二维码就会包含IP地址了。

问题3：扫描二维码后无法访问
可能原因：

扫描的手机没有连接同一WiFi
后端服务停止了
防火墙阻止了访问
解决方法：

确认扫描的手机连接同一WiFi
检查后端服务是否还在运行
检查防火墙设置
问题4：能访问列表，但看不到图片
可能原因：

图片路径配置问题
后端图片服务未启动
解决方法：

检查浏览器控制台（F12）是否有错误
确认图片文件存在于 backend/uploads/ 目录
检查后端日志
方法二：内网穿透（远程访问）
如果你想让不在同一WiFi的手机也能扫描，需要使用内网穿透。

使用 ngrok（推荐）
第1步：安装 ngrok
访问 https://ngrok.com/
注册账号
下载并安装 ngrok
第2步：启动穿透
穿透前端：

ngrok http 5173
你会看到：

Forwarding  https://abc123.ngrok.io -> http://localhost:5173
穿透后端：

ngrok http 8080
你会看到：

Forwarding  https://xyz789.ngrok.io -> http://localhost:8080
第3步：修改前端配置
修改 
request.js
：

const request = axios.create({
  baseURL: 'https://xyz789.ngrok.io/api',  // 使用后端的ngrok地址
  timeout: 10000
})
第4步：重启前端
cd frontend
npm run dev
第5步：访问和扫描
在任何设备的浏览器输入：https://abc123.ngrok.io/public-relics
点击文物查看详情
用手机扫描二维码
任何网络下的手机都能访问！
注意：

ngrok免费版每次启动URL会变化
速度可能较慢
适合临时演示
方法三：部署到服务器（正式使用）
如果要正式使用，建议部署到云服务器。

简要步骤
购买云服务器（阿里云、腾讯云等）
购买域名（可选但推荐）
部署后端：
mvn clean package
# 上传 target/*.jar 到服务器
java -jar cultural-relics.jar
部署前端：
npm run build
# 上传 dist/ 目录到服务器
# 使用 Nginx 托管
配置域名和SSL
访问：https://your-domain.com

方法	难度	成本	访问范围	速度	推荐场景
局域网IP	⭐ 简单	免费	同一WiFi	很快	开发测试

服务器部署	⭐⭐⭐ 复杂	需要服务器	任何网络	很快	正式使用
