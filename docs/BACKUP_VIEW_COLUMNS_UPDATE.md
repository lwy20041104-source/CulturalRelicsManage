# 数据备份列表显示列更新

## 更新日期
2026-05-09

## 更新内容

在数据备份管理界面的列表中添加了两个新的显示列：
1. **描述列** - 对应数据库表 `sys_backup` 的 `description` 字段
2. **错误信息列** - 对应数据库表 `sys_backup` 的 `error_message` 字段

## 修改的文件

### 1. 前端视图文件
**文件路径**: `frontend/src/views/BackupView.vue`

#### 修改内容
在表格中添加了两个新列，位于"创建时间"列之后、"操作"列之前：

```vue
<!-- 描述列 -->
<el-table-column 
  prop="description" 
  :label="$t('backup.description')" 
  min-width="200" 
  show-overflow-tooltip 
/>

<!-- 错误信息列 -->
<el-table-column 
  :label="$t('backup.errorMessage')" 
  min-width="200" 
  show-overflow-tooltip
>
  <template #default="scope">
    <span v-if="scope.row.errorMessage" style="color: #F56C6C;">
      {{ scope.row.errorMessage }}
    </span>
    <span v-else style="color: #909399;">—</span>
  </template>
</el-table-column>
```

#### 列的特性

**描述列**：
- 字段：`description`
- 标签：使用国际化 `$t('backup.description')`
- 最小宽度：200px
- 溢出提示：启用（`show-overflow-tooltip`）
- 显示内容：直接显示描述文本

**错误信息列**：
- 字段：`errorMessage`
- 标签：使用国际化 `$t('backup.errorMessage')`
- 最小宽度：200px
- 溢出提示：启用（`show-overflow-tooltip`）
- 显示逻辑：
  - 如果有错误信息：显示红色文本（#F56C6C）
  - 如果没有错误信息：显示灰色占位符"—"（#909399）

## 列的显示顺序

更新后的表格列顺序如下：

1. 备份名称（backupName）
2. 备份类型（backupType）
3. 状态（backupStatus）
4. 文件大小（fileSize）
5. 是否加密（isEncrypted）
6. 创建人（createdBy）
7. 创建时间（createdTime）
8. **描述（description）** ✨ 新增
9. **错误信息（errorMessage）** ✨ 新增
10. 操作（action）

## 数据库字段映射

| 前端显示列 | 数据库字段 | 数据类型 | 说明 |
|-----------|-----------|---------|------|
| 描述 | description | VARCHAR | 备份的描述信息 |
| 错误信息 | error_message | TEXT | 备份失败时的错误信息 |

## 显示效果

### 描述列
- 正常显示备份的描述文本
- 如果描述过长，会显示省略号，鼠标悬停时显示完整内容
- 如果没有描述，显示空白

### 错误信息列
- **有错误信息时**：
  - 文本颜色：红色（#F56C6C）
  - 显示完整的错误信息
  - 如果错误信息过长，会显示省略号，鼠标悬停时显示完整内容

- **无错误信息时**：
  - 显示灰色占位符"—"（#909399）
  - 表示该备份没有错误

## 使用场景

### 描述列
- 查看备份的用途和说明
- 区分不同备份的目的
- 帮助管理员快速识别备份内容

**示例**：
```
"系统升级前的完整备份"
"每日自动备份"
"数据迁移前的安全备份"
```

### 错误信息列
- 快速识别失败的备份
- 查看备份失败的具体原因
- 帮助排查和解决备份问题

**示例**：
```
"备份文件未创建或文件大小为0"
"数据库连接失败"
"磁盘空间不足"
"Java原生备份失败: Connection timeout"
```

### 1. 提升可追溯性

- 加快问题定位和解决速度

### 3. 增强用户体验
- 信息更加完整和透明
- 减少用户的困惑和疑问
- 提高系统的专业性

## 注意事项

### 1. 性能考虑
- 新增列不影响查询性能（字段已存在于数据库）
- 表格渲染性能正常（使用虚拟滚动）

### 2. 数据安全
- 错误信息可能包含敏感信息（如数据库连接信息）
- 建议在生产环境中过滤敏感信息
- 或者限制错误信息的查看权限

### 3. 国际化
- 已添加中英文翻译
- 如需支持其他语言，需要在对应的语言文件中添加翻译

### 4. 样式一致性
- 错误信息使用 Element Plus 的危险色（#F56C6C）
- 占位符使用 Element Plus 的信息色（#909399）
- 保持与系统整体风格一致
