### 4.2 验证数据库持久化
```sql
-- 查看最新的会话
SELECT * FROM ai_chat_session ORDER BY id DESC LIMIT 1;

-- 查看最新的消息
SELECT * FROM ai_chat_message ORDER BY id DESC LIMIT 5;

-- 查看最新的查询结果
SELECT * FROM ai_query_result ORDER BY id DESC LIMIT 10;

-- 验证数据完整性
SELECT 
    acm.id as message_id,
    acm.result_count as expected_count,
    COUNT(aqr.id) as actual_count,
    CASE 
        WHEN acm.result_count = COUNT(aqr.id) THEN '✅ 一致'
        ELSE '❌ 不一致'
    END as status
FROM ai_chat_message acm
LEFT JOIN ai_query_result aqr ON acm.id = aqr.message_id
WHERE acm.message_type = 'ai'
GROUP BY acm.id
ORDER BY acm.id DESC
LIMIT 5;
```

**预期结果：**
- ✅ ai_chat_session表有新记录
- ✅ ai_chat_message表有用户消息和AI回复
- ✅ ai_query_result表有查询结果详情
- ✅ result_count与实际保存的结果数量一致

## 七、故障排查清单

### 7.1 博物馆列表加载失败
**可能原因：**
- 数据库表未创建
- SecurityConfig未配置匿名访问
- 后端服务未重启

**解决方案：**
1. 执行museum_tables.sql
2. 检查SecurityConfig是否包含`.antMatchers(HttpMethod.GET, "/museums/active").permitAll()`
3. 重启后端服务

### 7.2 借展单位字段为空
**可能原因：**
- 登录时没有保存博物馆名称
- sessionStorage中没有museumName
- 前端服务未重启

**解决方案：**
1. 检查sessionStorage：`console.log(sessionStorage.getItem('museumName'))`
2. 重新登录
3. 重启前端服务

### 7.3 user_museum表没有新增记录
**可能原因：**
- 前端没有发送museumId
- 后端没有接收到museumId
- 数据库插入失败

**解决方案：**
1. 查看浏览器Network标签，检查请求Payload
2. 查看后端日志，确认是否输出调试信息
3. 手动执行SQL验证

### 7.4 AI查询结果没有保存
**可能原因：**
- 数据库表未创建
- 用户未登录
- 查询失败

**解决方案：**
1. 执行ai_chat_tables.sql
2. 确认用户已登录
3. 查看后端日志，检查是否有异常
