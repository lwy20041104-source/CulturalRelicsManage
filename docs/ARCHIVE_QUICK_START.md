# 数字化档案管理 - 快速开始指南

## 5分钟快速上手

### 第一步：初始化数据库

```bash
# 连接到MySQL数据库
mysql -u root -p

# 选择数据库
use your_database_name;

# 执行建表脚本
source backend/sql/digital_archive_tables.sql;
```

或者直接执行：
```bash
mysql -u root -p your_database_name < backend/sql/digital_archive_tables.sql
```

### 第二步：启动后端服务

```bash
cd backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动

### 第三步：启动前端服务

```bash
cd frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

前端服务将在 `http://localhost:5173` 启动

### 第四步：访问系统

1. 打开浏览器访问：`http://localhost:5173`
2. 使用管理员账号登录
3. 在左侧菜单找到"📚 档案管理"
4. 开始使用档案管理功能

## 功能演示

### 创建第一个档案

1. **进入档案管理**
   - 点击左侧菜单"📚 档案管理"

2. **新建档案**
   - 点击"新建档案"按钮
   - 选择一个文物（必填）
   - 填写档案标题，如："青铜器完整档案"
   - 选择档案类型：完整档案
   - 填写档案描述（可选）
   - 点击"确定"

3. **查看档案**
   - 档案创建成功后，会显示在列表中
   - 状态为"草稿"
   - 档案编号自动生成，如：AR-2024-001

### 上传文档

1. **进入档案详情**
   - 在档案列表中点击"详情"按钮

2. **上传文档**
   - 点击"上传文档"按钮
   - 选择文档类型：
     - 鉴定报告
     - 修复记录
     - 研究论文
     - 证书
     - 照片
     - 其他
   - 填写文档说明（可选）
   - 选择文件（支持PDF、Word、Excel、图片等）
   - 点击"上传"

3. **查看文档**
   - 上传成功后，文档会显示在"档案文档"列表中
   - 可以点击"预览"查看文档
   - 可以点击"删除"删除文档（仅草稿状态）

### 发布档案

1. **完善档案内容**
   - 确保已上传必要的文档
   - 检查档案信息是否完整

2. **发布**
   - 在详情页点击"发布"按钮
   - 确认发布操作
   - 档案状态变为"已发布"
   - ⚠️ 注意：发布后无法编辑

### 导出档案

1. **导出PDF**
   - 在列表页或详情页点击"导出PDF"
   - 系统自动生成PDF文档
   - 浏览器自动下载文件
   - 文件名格式：`档案_AR-2024-001_20240424100000.pdf`

2. **导出Word**
   - 在列表页或详情页点击"导出Word"
   - 系统自动生成Word文档
   - 浏览器自动下载文件
   - 文件名格式：`档案_AR-2024-001_20240424100000.docx`

3. **导出内容包含**
   - 档案基本信息
   - 关联文物信息
   - 档案文档清单
   - 导出时间戳

### 归档

1. **归档已发布的档案**
   - 在列表页点击"更多" → "归档"
   - 或在详情页点击"归档"按钮
   - 确认归档操作
   - 档案状态变为"已归档"

2. **归档后的档案**
   - 仍可查看
   - 仍可导出
   - 无法编辑
   - 无法删除

## 常见操作

### 搜索档案

在档案列表页：
- **按档案编号搜索**：输入档案编号，如"AR-2024-001"
- **按档案类型筛选**：选择完整档案、基础档案、图片档案、文档档案
- **按状态筛选**：选择草稿、已发布、已归档
- 点击"搜索"按钮

### 编辑档案

⚠️ 只有草稿状态的档案可以编辑

1. 在列表页点击"编辑"按钮
2. 修改档案信息
3. 点击"确定"保存
4. 系统自动增加版本号

### 删除档案

⚠️ 只有草稿状态的档案可以删除

1. 在列表页点击"更多" → "删除"
2. 确认删除操作
3. 档案及其所有文档将被永久删除

### 查看历史记录

在档案详情页：
- 滚动到"操作历史"部分
- 查看时间线展示的所有操作记录
- 包含：操作类型、操作人、操作时间、操作内容

## 文件上传限制

- **支持的文件格式**：
  - 文档：PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX
  - 图片：JPG、JPEG、PNG、GIF、BMP
  - 其他：TXT、ZIP、RAR
  
- **文件大小限制**：单个文件不超过 50MB

- **文件存储位置**：`backend/uploads/archives/`

## 档案编号规则

- **格式**：`AR-年份-序号`
- **示例**：
  - AR-2024-001
  - AR-2024-002
  - AR-2025-001
- **说明**：
  - AR = Archive（档案）
  - 年份：4位数字
  - 序号：3位数字，每年从001开始

## 状态流转

```
草稿 (draft)
  ↓ [发布]
已发布 (published)
  ↓ [归档]
已归档 (archived)
```

- **草稿**：可编辑、可删除、可上传文档
- **已发布**：只读、可归档、可导出
- **已归档**：只读、可导出

## 故障排除

### 问题1：无法上传文档

**可能原因**：
- 档案状态不是"草稿"
- 文件大小超过限制
- 文件格式不支持

**解决方法**：
- 检查档案状态
- 压缩文件或分割上传
- 转换文件格式

### 问题2：导出失败

**可能原因**：
- 档案数据不完整
- 服务器字体缺失（中文字体）

**解决方法**：
- 检查档案数据
- 安装中文字体（STSong-Light）

### 问题3：菜单中找不到"档案管理"

**可能原因**：
- 用户权限不足
- 前端路由未配置

**解决方法**：
- 检查用户角色权限
- 确认路由配置正确
