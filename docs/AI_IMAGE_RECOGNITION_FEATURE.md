# AI图像识别功能 - 文物自动分类

## 功能概述

AI图像识别功能可以自动分析上传的文物图片，智能识别文物类型并提供分类建议，大大提高文物录入的效率和准确性。

## 功能特点

### 1. 智能分类识别
- **主要分类推荐**：AI会给出最可能的文物分类，并显示置信度
- **备选分类**：提供其他可能的分类选项供用户选择
- **一键应用**：点击即可将识别结果应用到表单

### 2. 多维度分析
- **材质识别**：根据图片特征推测文物材质（青铜、瓷器、玉石等）
- **年代推测**：基于颜色和特征推测可能的历史年代
- **特征提取**：显示识别到的关键视觉特征

### 3. 支持的分类
系统支持识别以下10种文物分类：
1. 青铜器
2. 陶瓷器
3. 书画
4. 玉器
5. 金银器
6. 碑帖
7. 钱币
8. 服饰
9. 佛像
10. 杂项

## 使用方法

### 步骤1：上传图片
在添加或编辑文物时，先上传文物图片。

### 步骤2：启动AI识别
点击分类选择框旁边的"AI识别"按钮。

### 步骤3：查看识别结果
- 系统会显示识别结果，包括：
  - 推荐分类及置信度
  - 其他可能的分类
  - 建议的年代和材质
  - 识别到的特征

### 步骤4：应用结果
- 点击"应用此分类"按钮，自动填充分类字段
- 也可以点击"应用"按钮填充年代或材质字段
- 或者手动选择其他备选分类

## 技术实现

### 后端架构

#### 1. 服务接口
```java
public interface RelicImageRecognitionService {
    ImageRecognitionResult recognizeRelic(MultipartFile imageFile);
    ImageRecognitionResult recognizeRelicByUrl(String imageUrl);
}
```

#### 2. 识别算法
当前实现基于图像特征分析：
- **颜色分析**：提取主色调和颜色分布
- **亮度分析**：计算平均亮度
- **规则匹配**：基于颜色特征匹配文物类型

**示例规则**：
- 绿色/青色 → 青铜器或玉器
- 白色/灰色 → 陶瓷器或碑帖
- 黄色/金色 → 金银器或钱币
- 棕色/褐色 → 书画或服饰

#### 3. API端点
- `POST /api/relic-recognition/recognize` - 上传图片识别
- `POST /api/relic-recognition/recognize-by-url` - 通过URL识别

### 前端实现

#### 1. 组件集成
在 `RelicsView.vue` 中集成AI识别功能：
- AI识别按钮（分类选择框旁边）
- AI识别对话框（显示识别结果）
- 结果应用逻辑

#### 2. 用户交互
- 加载动画：识别过程中显示加载状态
- 结果展示：卡片式展示识别结果
- 快速应用：一键应用识别结果

## 注意事项

1. **识别准确性**：当前实现基于简单规则，准确率有限，建议人工复核
2. **图片质量**：清晰、光线充足的图片识别效果更好
3. **文件大小**：建议图片大小在5MB以内
4. **支持格式**：JPG、PNG、WEBP等常见图片格式

## 性能优化

### 图片采样
为提高性能，识别算法采用采样分析：
- 每隔10个像素采样一次
- 减少计算量，提高响应速度

### 缓存机制
可以考虑添加识别结果缓存：
- 相同图片不重复识别
- 使用Redis缓存识别结果
