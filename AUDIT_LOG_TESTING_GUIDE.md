# 审计日志功能测试指南

## 🎯 测试目的

验证审计日志的数据对比功能是否正常工作，包括：
- 操作前后数据的完整记录
- 字段级别的变更对比
- 变更字段的高亮显示

---

## ⚠️ 前置条件

### 1. 执行数据库脚本（必须）

```bash
# 备份数据库
mysqldump -u root -p cultural_relics > backup_before_audit_log.sql

# 执行增强脚本
mysql -u root -p cultural_relics < backend/sql/audit_log_enhancement.sql

# 验证表结构
mysql -u root -p cultural_relics
```

在MySQL中执行以下验证命令：

```sql
-- 检查sys_operation_log表是否有新字段
DESC sys_operation_log;

-- 应该看到以下新字段：
-- before_data
-- after_data
-- changed_fields
-- resource_type
-- resource_id
-- request_method
-- request_url
-- browser
-- os
-- execution_time
-- 等等...

-- 检查sys_data_change_detail表是否创建
DESC sys_data_change_detail;
```

```bash

cd backend

# 启动前端（新终端）

```

## 🐛 常见问题排查

### 问题1：详情页没有显示数据对比

**可能原因**：
1. 数据库脚本未执行
2. 旧的日志记录没有这些字段的数据

**解决方案**：
1. 确认数据库脚本已执行：
   ```sql
   DESC sys_operation_log;
   -- 检查是否有before_data, after_data, changed_fields字段
   ```

2. 执行新的操作生成新的日志记录
3. 查看新生成的日志记录

### 问题2：变更字段显示为英文字段名

**可能原因**：
字段标签映射未配置

**解决方案**：
检查`AuditLogUtil.java`中是否有对应资源类型的字段标签映射方法。

### 问题3：数据对比区域为空

**可能原因**：
1. 操作类型不是修改/删除
2. beforeData或afterData为空

**解决方案**：
1. 只有修改和删除操作才会有数据对比
2. 新增操作只有afterData
3. 查询操作不记录数据变更

### 问题4：JSON格式显示错误

**可能原因**：
数据不是有效的JSON格式

**解决方案**：
检查后端的`AuditLogUtil.toJson()`方法是否正确转换对象为JSON。
